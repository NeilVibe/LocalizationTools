## How to Add New Issues

Use format: `BUG-XXX` or `UI-XXX`

```markdown
### BUG-XXX: [Brief Title]
- **Status:** [ ] Open
- **Priority:** Critical/High/Medium/Low
- **Reported:** YYYY-MM-DD
- **Component:** [Area affected]

**Problem:** [What's wrong]

**Expected:** [What should happen]

**Files:** [Likely locations]
```

## Categories

- **BUG-XXX** - Functional bugs, crashes, errors
- **UI-XXX** - UI/UX issues, styling, layout
- **PERF-XXX** - Performance issues
- **SEC-XXX** - Security issues
