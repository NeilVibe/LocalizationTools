## 1. 시스템 개요

LocaNext는 게임 로컬라이제이션(번역) 관리 도구입니다. LAN 서버 모드는 사내 네트워크에서 팀원들이 동시에 번역 데이터를 편집할 수 있도록 하는 기능입니다.

### 아키텍처

```
관리자 PC (서버 모드)                    팀원 PC (클라이언트 모드)
┌──────────────────────────────┐       ┌──────────────────────────────┐
│ LocaNext (전체 설치)          │       │ LocaNext (일반 설치)          │
│ ├── Python 백엔드 (로컬)      │       │ ├── Python 백엔드 (로컬)      │
│ ├── MegaIndex (로컬)          │       │ ├── MegaIndex (로컬)          │
│ ├── FAISS 검색 (로컬)         │       │ ├── FAISS 검색 (로컬)         │
│ ├── 오디오/이미지 (로컬)       │       │ ├── 오디오/이미지 (로컬)       │
│ └── PostgreSQL 서버 ◄────────────────►│ └── PG 클라이언트 → 관리자 PG  │
│     ├── 모든 번역 데이터       │  LAN  │                               │
│     ├── 사용자 계정            │ (TLS) │ 설정:                         │
│     ├── 세션/잠금              │       │   서버: 192.168.x.x:5432     │
│     └── TM (번역 메모리)       │       └──────────────────────────────┘
└──────────────────────────────┘
```

**핵심 원칙:** 데이터베이스 작업만 네트워크를 통해 공유됩니다. 각 PC는 자체 백엔드를 실행하여 무거운 작업(인덱싱, 검색, 파일 파싱)을 로컬에서 처리합니다.

## 2. 네트워크 보안

### 개방 포트

| 포트 | 프로토콜 | 용도 | 접근 범위 |
|------|----------|------|-----------|
| 5432 | TCP (TLS) | PostgreSQL 데이터베이스 | 사내 LAN 서브넷만 (/24) |
| 8888 | TCP | FastAPI 백엔드 (관리자 대시보드) | localhost만 (127.0.0.1) |
| 5174 | TCP | 관리자 대시보드 UI | localhost만 (127.0.0.1) |

### 네트워크 제한

- **인터넷 노출 없음** — 모든 포트는 사내 LAN에서만 접근 가능
- **pg_hba.conf** — 사무실 서브넷(예: 192.168.x.0/24)만 허용
- **Windows 방화벽** — 포트 5432 인바운드 규칙: LAN 범위만 허용
- **클라우드 의존성 없음** — 외부 서비스 연결 없음

### TLS/SSL 암호화

- PostgreSQL 연결에 **자체 서명 SSL 인증서** 적용
- 인증서 유효기간: 10년
- 암호화 알고리즘: TLS 1.2+ (OpenSSL 제공)
- **pg_hba.conf**에 `hostssl` 규칙 적용, `hostnossl reject`로 비암호화 연결 차단
- 오프라인 환경에서도 동작 (인터넷 불필요)

## 3. 인증 및 권한

### 인증 체계 (이중 보안)

| 계층 | 방식 | 설명 |
|------|------|------|
| **앱 레벨** | JWT (HS256) + bcrypt | 사용자별 로그인, 토큰 기반 인증 (만료: 60분) |
| **DB 레벨** | scram-sha-256 | PostgreSQL 표준 인증 (서비스 계정) |

### IP 기반 관리자 인증 (IP-Lock Admin)

- 서버 설치 시 해당 PC가 **원본 관리자(Origin Admin)**로 자동 등록
- localhost(127.0.0.1)에서 전용 POST 엔드포인트(`/api/origin-admin-token`) 호출 시 관리자 JWT 발급
- 원격 접속(팀원 PC) = **반드시 사용자 이름/비밀번호 필요**
- 초기 관리자 계정(admin/admin123) 생성 후 첫 로그인 시 비밀번호 변경 필수 (`must_change_password`)

### 비밀번호 정책

- **bcrypt** 해싱 (기본 12 라운드, 환경 변수로 변경 가능) — 업계 표준
- 첫 로그인 시 비밀번호 변경 강제 (`must_change_password`)
- 로그인 시도 제한: IP당 15분 내 5회 실패 → 잠금
- 비밀번호 평문 저장 없음

### 역할 관리

| 역할 | 권한 |
|------|------|
| **user** | 번역 데이터 읽기/쓰기, 자기 파일 관리 |
| **admin** | 사용자 관리, 시스템 설정, 모든 데이터 접근 |
| **superadmin** | 관리자 권한 부여, 시스템 전체 관리 |

## 4. 데이터 보안

### 데이터 분류

| 데이터 유형 | 분류 | 저장 위치 |
|------------|------|-----------|
| 게임 번역 문자열 | **사내 기밀 (NDA 적용)** | PostgreSQL (관리자 PC) |
| 사용자 계정 정보 | **개인정보** | PostgreSQL (bcrypt 해싱) |
| 오디오/이미지 파일 | **사내 기밀** | 각 PC 로컬 저장 |
| 로그/감사 기록 | **내부용** | PostgreSQL |

### 감사 로그 (Audit Logging)

- 모든 로그인 시도 기록 (성공/실패, IP 주소, 시간)
- 데이터 변경 이력 추적
- 관리자 작업 로그 (사용자 생성/삭제/권한 변경)
- 로그 보존 기간: 90일

### 백업

- `db_manager.sh` 스크립트로 수동 백업 가능
- 백업 파일: PostgreSQL `pg_dump` 형식
- 권장: 관리자가 매일 백업 실행

## 7. 설치 및 운영 절차

### 관리자 설치 (1회)

```
1. LocaNext 설치 (관리자 빌드)
2. PostgreSQL 설치
3. python setup_pg_server.py 실행 (자동 설정)
   → DB 생성, SSL 인증서 생성, 방화벽 설정
   → 관리자 PC = IP-Lock Admin (자동 관리자 권한)
4. http://localhost:5174 에서 팀원 계정 생성
5. 팀원에게 서버 IP 주소 전달
```

### 팀원 설치

```
1. LocaNext 설치 (일반 빌드 — PostgreSQL 불필요)
2. 설정 → 서버 연결 → 관리자 IP:5432 입력
3. 연결 테스트 → 저장 → 재시작
4. 관리자가 만든 계정으로 로그인
```

### 관리자 빌드 vs 일반 빌드

| 항목 | 관리자 빌드 | 일반 빌드 (Light) |
|------|------------|-------------------|
| LocaNext 앱 | ✅ | ✅ |
| PostgreSQL | ✅ (번들 또는 별도 설치) | ❌ 불필요 |
| 설정 스크립트 | ✅ setup_pg_server.py | ❌ 불필요 |
| 관리자 대시보드 | ✅ (localhost 접속) | ❌ 접근 불가 |
| 서버 연결 설정 | 자동 (localhost) | 수동 (IP 입력) |

## 9. 결론

LocaNext LAN 서버 모드는 **표준 기업 내부 도구**와 동일한 보안 수준을 제공합니다:

- **업계 표준 인증**: JWT + bcrypt + scram-sha-256
- **네트워크 격리**: 사내 LAN만, 인터넷 노출 없음
- **암호화**: TLS/SSL (자체 서명 인증서)
- **접근 제어**: IP 기반 관리자 + 역할 기반 권한
- **감사**: 전체 로그인/작업 기록

외부 클라우드 서비스 의존성이 없으며, 모든 데이터는 사내 네트워크 내에서만 유통됩니다.

## 10. 비밀 키 관리

| 키 | 저장 위치 | 생성 방식 | 교체 주기 |
|----|-----------|-----------|-----------|
| JWT 서명 키 (SECRET_KEY) | `server-config.json` (0600 권한) | 서버 설정 시 자동 생성 (secrets.token_urlsafe) | 수동 교체 |
| PG 서비스 계정 비밀번호 | `server-config.json` (0600 권한) | 서버 설정 시 자동 생성 (24자 URL-safe) | 수동 교체 |
| SSL 인증서 | PG 데이터 디렉토리 (0600 권한) | 자체 서명 (OpenSSL, 10년 유효) | 만료 시 재생성 |

## 11. 개인정보보호법 (PIPA) 관련

본 시스템은 **사내 로컬라이제이션 도구**로서 외부 사용자 데이터를 처리하지 않습니다. 사내 사용자 계정 정보(이름, 이메일)는 사내 보안 정책에 따라 관리되며, 모든 데이터는 사내 네트워크 내에서만 유통됩니다.
