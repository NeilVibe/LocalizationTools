Memory & Docs

Critical

Record while working

Decisions, findings, context. During, not after.

Keep it fresh

Stale context = wrong AI output. Update actively.

Purge aggressively

Delete outdated memory. Less is more accurate.

Self-improve

Promote patterns into rules each cycle.

AI remembers nothing between sessions. Your docs ARE its brain. Bad docs = blind AI.

Runs parallel to every step.
Not a phase. A discipline.

Key Principles

01 Small increments. Big changes = big failures.
02 Verify everything. AI hallucinates APIs and libraries.
03 Scope discipline. AI over-engineers. Say "only what I asked."
04 Root cause first. Why, then how.
05 Security is mandatory. OWASP before every ship.
06 Proof over promises. Tests, logs, screenshots.

Rich Logging

Critical

Logs are for the AI, not you

Humans skim logs. AI reads every line. Rich logs give AI full execution context.

Log every decision point

What ran, what was skipped, why. Every branch, every fallback, every value.

AI fixes 100x better with context

A stack trace alone is guesswork. A detailed execution log is a roadmap.

Expose limitations

Logs reveal what the code can't do. That's how you find what to build next.

Always make AI write code with rich logging. This is the single biggest force multiplier for AI-assisted debugging.

Baked into every function.
Not an afterthought. A requirement.

What to Log

IN Function entry + input values + state
DO Each step, branch taken, values computed
OUT Result, duration, success/failure, why
ERR Full context: what was attempted, what failed, what state was left
