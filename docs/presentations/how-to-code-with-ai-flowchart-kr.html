AI와 함께 코딩하는 법

아이디어에서 배포까지, 전체 개발 워크플로우

10 STEPS + 2 DISCIPLINES
Phase 1 — 설계
01

컨텍스트 수집

코드를 건드리기 전에 기존 코드, 문서, 제약 조건을 먼저 읽는다. AI는 내가 주는 맥락만큼만 똑똑하다.

02

계획 수립

작업을 작은 단위로 나눈다. 범위, 접근 방식, 완료 기준을 정한다. 코딩 전에 리뷰를 받는다.

03

계획 리뷰

가정을 의심한다. 엣지 케이스를 검증한다. 잘못된 방향은 2시간 코딩 후가 아니라 지금 잡는다.

Phase 2 — 구현
04

코드 작성

작은 단위로 구현한다. AI가 만들어낸 가짜 API, 과도한 설계, 범위 초과를 경계한다.

05

코드 리뷰

AI 결과물을 전부 검토한다. import가 실제로 존재하는지, 요청한 것만 했는지 확인한다.

Phase 3 — 검증
06

테스트

직접 실행한다. 자동 테스트 + 수동 확인. "될 거야"를 절대 믿지 않는다. 증거를 확보한다.

07

로그 확인

종료 코드만 보지 말고 전체 출력을 읽는다. 예상이 아닌 실제 결과를 파악한다.

08

버그 수정

근본 원인을 먼저 파악한다. 작고 정확한 수정. 땜질하지 말고 왜 깨졌는지 이해한다.

전부 통과할 때까지 04번으로 반복
Phase 4 — 강화 & 배포

09 — 보안 검증

배포 전 필수 점검. 예외 없음.

인젝션 인증/인가 시크릿 노출 입력값 검증 의존성 점검 OWASP Top 10
10

최종 리뷰

전체 점검. 계획과 일치하는가? 범위가 맞는가? 불필요한 코드 정리. 프로덕션 전 마지막 기회.

배포

메모리 & 문서

필수

작업하면서 기록한다

결정 사항, 발견한 것, 맥락. 끝나고가 아니라 하면서.

항상 최신 상태로 유지

낡은 맥락 = 잘못된 AI 결과. 적극적으로 갱신한다.

과감하게 정리한다

오래된 문서와 메모리를 삭제. 적을수록 정확하다.

스스로 개선한다

매 사이클에서 배운 패턴을 규칙으로 승격시킨다.

AI는 세션 사이에 아무것도 기억 못 한다. 문서가 곧 AI의 뇌다. 문서가 부실하면 AI도 부실하다.

모든 단계에 병렬로 진행.
단계가 아니라 습관.

핵심 원칙

01 작게 나눠서. 큰 변경 = 큰 실패.
02 전부 검증한다. AI는 없는 API를 만들어낸다.
03 범위를 지킨다. AI는 과하게 만든다. "요청한 것만."
04 근본 원인 먼저. 왜 깨졌는지 이해하고 고친다.
05 보안은 선택이 아니다. 배포 전 OWASP 필수.
06 증거로 말한다. 테스트, 로그, 스크린샷.

상세 로깅

필수

로그는 사람이 아니라 AI를 위한 것

사람은 로그를 훑어본다. AI는 한 줄 한 줄 다 읽는다. 상세한 로그가 AI에게 실행 맥락을 준다.

모든 분기점을 기록한다

뭐가 실행됐고, 뭐가 건너뛰어졌고, 왜 그랬는지. 모든 분기, 모든 값.

AI는 맥락이 있으면 100배 잘 고친다

스택 트레이스만으로는 추측이다. 상세한 실행 로그는 정확한 지도다.

한계를 드러낸다

로그가 코드의 한계를 보여준다. 다음에 뭘 만들지 알려주는 신호다.

AI에게 항상 상세한 로깅이 포함된 코드를 작성하게 한다. AI 디버깅의 가장 큰 생산성 배율기.

모든 함수에 내장한다.
나중에가 아니라 처음부터.
