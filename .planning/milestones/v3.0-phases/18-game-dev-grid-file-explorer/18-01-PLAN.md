  <behavior>
    - test_scan_folder: Given a directory with XML files, returns tree structure with folder names, file names, entity counts
    - test_scan_folder_empty: Given an empty directory, returns tree with empty folders/files lists
    - test_scan_folder_nested: Given nested directories, returns recursive tree up to max_depth
    - test_path_traversal_rejected: Given a path outside allowed base, raises ValueError
    - test_detect_columns: Given an XML file, returns list of metadata column keys present (Key, StrKey, KnowledgeKey, etc.)
    - test_detect_editable_attrs: Given entity tag name, returns list of editable attribute names from EDITABLE_ATTRS map
    - test_update_attribute: Given an XML file path, entity index, attr name, and new value, updates the attribute and saves
    - test_br_tag_round_trip: Given text with <br/> tags, saves to XML, re-reads, and br-tags are preserved correctly
    - test_update_attribute_invalid_index: Given out-of-range entity index, returns False without crashing
  </behavior>
  <action>
  Create schemas first (server/tools/ldm/schemas/gamedata.py):
  - FolderNode: name, path, folders (list of FolderNode), files (list of FileNode)
  - FileNode: name, path, size, entity_count
  - FolderTreeResponse: root FolderNode + base_path
  - ColumnHint: key, label, editable (bool)
  - FileColumnsResponse: columns (list of ColumnHint), editable_attrs (list of str)
  - GameDevSaveRequest: xml_path, entity_index, attr_name, new_value
  - GameDevSaveResponse: success, message

  Create GameDataBrowseService (server/tools/ldm/services/gamedata_browse_service.py):
  - scan_folder(root_path, max_depth=4): Recursively scan directory, return FolderNode tree. Only include .xml files. Use pathlib. Count entities via quick lxml parse (len(root) on each XML). Validate path is within allowed base using Path.resolve() + is_relative_to().
  - detect_columns(xml_path): Parse first entity element, return list of ColumnHint based on which attributes exist. Mark attributes in EDITABLE_ATTRS[entity_tag] as editable=True.
  - Use loguru logger (never print()).

  Create GameDataEditService (server/tools/ldm/services/gamedata_edit_service.py):
  - update_entity_attribute(xml_path, entity_index, attr_name, new_value): Parse XML with lxml, find entity by index under root, set attribute value, write back with encoding="UTF-8" and xml_declaration=True. Return True on success, False on invalid index.
  - CRITICAL br-tag handling: Pass user's text (containing <br/>) directly to element.set(). lxml auto-escapes to &lt;br/&gt; on disk. On re-read, lxml returns <br/> in the string. Do NOT pre-escape.
  - Validate xml_path is within allowed base directory before any file I/O.
  - Use loguru logger.

  Write tests FIRST (RED), then implement (GREEN).
  Use the mock gamedata from Phase 15 as fixtures: tests/mock_gamedata/ or create minimal XML fixtures inline with tmp_path.
