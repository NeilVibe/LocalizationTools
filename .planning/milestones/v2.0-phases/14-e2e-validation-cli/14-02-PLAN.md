**XML Fixtures (inline in the test file):**

1. TRANSLATOR_XML fixture (LocStr format):
```xml
<?xml version="1.0" encoding="utf-8"?>
<LocStr>
  <String StringId="ITEM_001" StrOrigin="Sword of Dawn" Str="" Desc="" DescOrigin="A legendary sword"/>
  <String StringId="ITEM_002" StrOrigin="Shield of Light" Str="" Desc="" DescOrigin="A holy shield"/>
  <String StringId="ITEM_003" StrOrigin="First line&lt;br/&gt;Second line" Str="" Desc="" DescOrigin=""/>
  <String StringId="ITEM_004" StrOrigin="Potion of Health" Str="" Desc="" DescOrigin="Restores HP"/>
  <String StringId="ITEM_005" StrOrigin="Ring of Power" Str="" Desc="" DescOrigin="Increases ATK"/>
</LocStr>
```

2. CORRECTIONS_XML fixture (source file for merge):
```xml
<?xml version="1.0" encoding="utf-8"?>
<LocStr>
  <String StringId="ITEM_001" StrOrigin="Sword of Dawn" Str="Dawn Sword KR" Desc="" DescOrigin="A legendary sword"/>
  <String StringId="ITEM_002" StrOrigin="Shield of Light" Str="Light Shield KR" Desc="" DescOrigin="A holy shield"/>
  <String StringId="ITEM_003" StrOrigin="First line&lt;br/&gt;Second line" Str="Line1&lt;br/&gt;Line2" Desc="" DescOrigin=""/>
  <String StringId="ITEM_004" StrOrigin="Potion of Health" Str="Health Potion KR" Desc="" DescOrigin="Restores HP"/>
</LocStr>
```

3. GAMEDEV_XML fixture (non-LocStr):
```xml
<?xml version="1.0" encoding="utf-8"?>
<GameData>
  <ItemList>
    <Item Key="item_001" Name="Sword" Attack="50" Defense="10"/>
    <Item Key="item_002" Name="Shield" Attack="5" Defense="80"/>
  </ItemList>
</GameData>
```

**Test class: TestTranslatorRoundTrip**

1. `test_translator_xml_roundtrip`:
   - Parse TRANSLATOR_XML with parse_xml_file -> (target_rows, target_meta)
   - Parse CORRECTIONS_XML with parse_xml_file -> (source_rows, _)
   - Assert target_meta["file_type"] == "translator"
   - Run TranslatorMergeService().merge_files(source_rows, target_rows, match_mode="strict")
   - Apply updated_rows to target_rows (update target_text where matched)
   - Build row dicts for ExportService (must have string_id, source_text, target_text keys)
   - Export to XML via ExportService().export_xml(rows, target_meta)
   - Re-parse exported XML with parse_xml_file
   - Assert ITEM_001 target == "Dawn Sword KR", ITEM_002 target == "Light Shield KR"
   - Assert ITEM_005 target is still empty (no correction source for it)

2. `test_translator_brtag_roundtrip`:
   - Same flow but focus on ITEM_003
   - After merge, ITEM_003 target should contain br-tag content
   - After export+re-parse, br-tags must still be present (not entity-doubled)

3. `test_translator_export_excel_columns`:
   - Parse TRANSLATOR_XML, export via ExportService().export_excel()
   - Assert output starts with b'PK' (valid zip/xlsx)
   - Assert len(output) > 0

4. `test_translator_export_text_format`:
   - Parse TRANSLATOR_XML, export via ExportService().export_text()
   - Decode output as UTF-8, split lines
   - Assert each line has tab-separated fields
   - Assert "ITEM_001" appears in output

**Test class: TestGameDevRoundTrip**

5. `test_gamedev_xml_roundtrip`:
   - Parse GAMEDEV_XML with parse_xml_file -> (rows, meta)
   - Assert meta["file_type"] == "gamedev"
   - Modify rows[0] to change Attack attribute from "50" to "99" (update target_text or extra_data.attributes)
   - Run GameDevMergeService().merge(GAMEDEV_XML.encode(), modified_rows)
   - Assert result.modified_attributes >= 1
   - Re-parse result.output_xml
   - Find item_001 and verify Attack="99"

**Test class: TestFileTypeDetection**

6. `test_file_type_detection_translator`: parse_xml_file(TRANSLATOR_XML) -> meta["file_type"] == "translator"
7. `test_file_type_detection_gamedev`: parse_xml_file(GAMEDEV_XML) -> meta["file_type"] == "gamedev"

Note: These tests import services directly. No server needs to be running. Use `from __future__ import annotations` at the top. If Model2Vec is not installed for fuzzy matching, the strict mode tests will still work (fuzzy is not tested in round-trip -- it requires model files).
