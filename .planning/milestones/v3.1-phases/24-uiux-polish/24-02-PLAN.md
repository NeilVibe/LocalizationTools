Run audit skill and harden skill mentally across ALL v3.0 components. Read each component, identify and fix issues in these categories:

**Accessibility (audit):**
- Add `aria-label` to icon-only buttons (close buttons, refresh buttons, navigation icons)
- Add `role="status"` or `aria-live="polite"` to loading indicators and error banners
- Ensure all custom buttons have visible `:focus` styles (2px solid var(--cds-focus))
- Add `aria-label` to CodexSearchBar input if missing
- Ensure MapCanvas has `role="img"` or appropriate ARIA for the canvas element
- Add `role="tablist"` to CodexPage `.codex-tabs` container and `role="tab"` + `aria-selected` to each tab button

**Error hardening (harden):**
- Review all `catch` blocks -- ensure error messages shown to users are human-readable, not raw HTTP status codes
- Add empty state messaging where missing (e.g., "No suggestions yet -- select a text entry to see AI suggestions")
- Ensure text overflow is handled in ALL text-displaying elements (add `overflow-wrap: break-word` where missing)
- Review MapTooltip positioning -- ensure it does not render off-screen

**UX copy (clarify):**
- Review loading text: use action-oriented text ("Loading suggestions..." not just "Loading...")
- Review error messages: be specific about what failed and what to try ("Failed to load entity list -- check your network connection" not just "Error")
- Review empty states: guide the user ("Select a file to see AI translation suggestions" not "No data")
- Review button labels: ensure they describe the action ("Open in Codex" is good, verify others)

Implement all findings directly. Fix the code AND document every finding.

**After completing all fixes, write a summary to `.planning/phases/24-uiux-polish/24-02-AUDIT-FINDINGS.md`** listing:
- Every issue found, grouped by category (Accessibility / Error Hardening / UX Copy)
- The component file where the issue was found
- The fix applied (one-line description)
- Issues found but intentionally skipped (with reason)

Format the audit findings file as a markdown table per category.

Prioritize fixes by impact:
1. Missing focus styles (breaks keyboard navigation)
2. Missing ARIA attributes (breaks screen readers)
3. Raw error messages (confuses users)
4. Missing empty states (leaves users lost)
5. Text overflow (breaks layouts)

Skip any changes that would conflict with Plan 01's modifications (PlaceholderImage, layout tabs -- those are already fixed).
