**FIX-05: AISuggestionsTab and NamingPanel loading state on debounce cancel**

Both components have the same pattern: the `$effect` cleanup function clears the debounce timer but does NOT clear the loading state. If a user rapidly navigates away before the debounce fires, `loading` stays `true` forever (infinite spinner).

In AISuggestionsTab.svelte, update the cleanup return in the $effect (around line 116-119):
```javascript
return () => {
  if (debounceTimer) clearTimeout(debounceTimer);
  if (abortController) abortController.abort();
  loading = false;  // <-- ADD THIS: Clear loading on cleanup
};
```

In NamingPanel.svelte, update the cleanup return in the $effect (around line 121-123):
```javascript
return () => {
  if (debounceTimer) clearTimeout(debounceTimer);
  if (abortController) abortController.abort();
  loading = false;  // <-- ADD THIS: Clear loading on cleanup
};
```

Both cleanup functions should also abort the controller to prevent stale responses from setting state after cleanup.

**FIX-06: QAInlineBadge click-outside and backdrop tabindex**

Two issues in QAInlineBadge.svelte:

1. The `handleClickOutside` function (lines 139-146) is defined but never used -- the backdrop `onclick` directly calls `closePopover()`. The handleClickOutside function can be removed as dead code since the backdrop approach works correctly.

2. The backdrop div (line 174-178) needs a `tabindex` attribute for keyboard accessibility so that Escape key handling works:
```svelte
<div
  class="qa-popover-backdrop"
  onclick={(e) => { e.stopPropagation(); closePopover(); }}
  onkeydown={(e) => e.key === "Escape" && closePopover()}
  tabindex="-1"
  role="presentation"
></div>
```

3. Remove the unused `handleClickOutside` function (lines 139-146) to avoid confusion.
