  <action>
    Create `testing_toolkit/run_api_tests.sh` — a bash script for overnight autonomous API test execution.

    **Features:**
    1. Pre-flight checks: verify server is running (localhost:8888/health), verify Python/pytest available
    2. Subsystem ordering: health -> auth -> projects -> folders -> files -> rows -> tm -> gamedata -> codex -> worldmap -> ai -> search -> qa -> merge -> export -> offline -> admin -> tools -> integration
    3. Per-subsystem execution: `pytest tests/api/test_{subsystem}.py -v --tb=short --timeout=120`
    4. Result capture: append pass/fail counts to a results file
    5. Summary report: at end, print total pass/fail/skip across all subsystems
    6. Subsystem filtering: `./run_api_tests.sh auth files` runs only those subsystems
    7. `./run_api_tests.sh all` runs everything (default)
    8. Timestamp each subsystem start/end for performance tracking
    9. Save full output to `testing_toolkit/api_test_results_{date}.log`
    10. Exit code: 0 if all pass, 1 if any fail

    **Error handling:**
    - If server not running, print clear instructions and exit 1
    - If a subsystem test file does not exist yet, SKIP it (don't fail — Wave 2 creates them)
    - If pytest crashes, capture the error and continue to next subsystem
    - Timeout per test: 120 seconds (some AI/search tests may be slow)

    Make the script executable (chmod +x).
  </action>

  </verify>
