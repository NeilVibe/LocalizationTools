# v3.5 WOW Showcase — Requirements

**Defined:** 2026-03-17
**Core Value:** Transform LocaNext from functional tool to visually stunning showcase with delightful interactions, immersive world visualization, and polished micro-interactions that make every click feel premium.

---

## Phase 47: XML Viewer WOW Polish

### REQ-47-01: Smart Attribute Highlighting
**Description:** Highlight individual attributes with semantic colors based on attribute type, replacing the flat full-line highlight with meaningful visual cues.
**Acceptance Criteria:**
- [ ] Cross-ref attrs (KnowledgeKey, FactionKey, RegionKey) show blue with underline cursor
- [ ] Editable attrs (CharacterName, CharacterDesc, Str, Desc) show green glow border
- [ ] Identity attrs (Key, StrKey, StringID) show gold/amber with subtle badge
- [ ] Media attrs (UITextureName, VoicePath, IconPath) show purple with media icon
- [ ] Numeric attrs (Level, HP, ATK, DEF) show cyan with monospace font
- [ ] Unknown/other attrs default to neutral gray highlight
- [ ] Color legend tooltip accessible via info icon in viewer header

### REQ-47-02: Hover Tooltips with Preview
**Description:** Show contextual tooltip on attribute hover with value preview, type info, and quick actions.
**Acceptance Criteria:**
- [ ] Hovering a cross-ref attr shows target entity name + type in tooltip (e.g., "Faction: Iron Brotherhood")
- [ ] Hovering a media attr shows thumbnail preview (image) or waveform icon (audio)
- [ ] Hovering an editable attr shows "Double-click to edit" hint
- [ ] Tooltip appears after 300ms delay, positioned to avoid viewport overflow
- [ ] Tooltip has subtle fade-in animation (150ms ease-out)
- [ ] Tooltip dismissed on mouse leave or scroll

### REQ-47-03: Smooth Expand/Collapse Animations
**Description:** XML tree nodes animate smoothly when expanding and collapsing, with staggered child reveal.
**Acceptance Criteria:**
- [ ] Expand animates height from 0 with 200ms ease-out transition
- [ ] Collapse animates height to 0 with 150ms ease-in transition
- [ ] Child nodes stagger in with 30ms delay per child on expand
- [ ] Chevron icon rotates 90 degrees smoothly during expand/collapse
- [ ] Large trees (50+ children) skip stagger and use batch reveal
- [ ] Animation respects prefers-reduced-motion media query

### REQ-47-04: Inline Attribute Search with Highlight
**Description:** Ctrl+F within the XML viewer highlights matching attributes across all visible nodes with scroll-to-match navigation.
**Acceptance Criteria:**
- [ ] Ctrl+F opens compact search bar at top of XML viewer
- [ ] Matching attribute names and values highlighted in yellow
- [ ] Up/Down arrows navigate between matches with smooth scroll
- [ ] Match count displayed (e.g., "3 of 12 matches")
- [ ] Search is case-insensitive by default with toggle for case-sensitive
- [ ] Escape closes search bar and clears highlights

## Phase 48: Fantasy World Map

### REQ-48-01: Parchment Map Aesthetic
**Description:** Transform the world map background from plain canvas to an aged parchment aesthetic with torn edges, ink stains, and compass rose.
**Acceptance Criteria:**
- [ ] Map background uses parchment texture with subtle grain noise
- [ ] Map edges have torn/burnt paper effect using CSS clip-path or SVG mask
- [ ] Compass rose SVG rendered in bottom-right corner with north indicator
- [ ] Map title rendered in fantasy serif font (e.g., Cinzel or similar)
- [ ] Subtle vignette darkening at edges for depth
- [ ] Background adapts to dark/light theme (sepia vs dark parchment)

### REQ-48-02: Region Polygons with Terrain Fill
**Description:** Replace simple region markers with filled polygons that convey terrain type through pattern fills and border styles.
**Acceptance Criteria:**
- [ ] Each region rendered as polygon with distinct fill pattern (forest=tree pattern, desert=dots, mountain=peaks)
- [ ] Region borders use hand-drawn stroke effect (rough.js or SVG filter)
- [ ] Region polygon opacity increases on hover (0.3 -> 0.6)
- [ ] Region name label positioned at polygon centroid with text shadow
- [ ] At least 5 terrain types supported: forest, desert, mountain, plains, water
- [ ] Polygons scale correctly during zoom (stroke width adjusts)

### REQ-48-03: Animated Terrain Icons
**Description:** Place animated SVG terrain icons (trees, mountains, waves) within region polygons for visual richness.
**Acceptance Criteria:**
- [ ] Forest regions show small tree icons with subtle sway animation
- [ ] Mountain regions show peak icons with shadow shift on time-of-day cycle
- [ ] Water regions show wave icons with gentle undulation animation
- [ ] Town/city markers show building icons with chimney smoke particles
- [ ] Icons scale with zoom level (hide below threshold, grow above)
- [ ] All animations pause when map is not visible (IntersectionObserver)

### REQ-48-04: Map Interaction Polish
**Description:** Map interactions feel premium with smooth transitions, contextual cursors, and haptic-like feedback.
**Acceptance Criteria:**
- [ ] Pan uses momentum scrolling with deceleration curve
- [ ] Zoom centers on cursor position with smooth interpolation (not step zoom)
- [ ] Double-click zooms in on region with fly-to animation (500ms ease-in-out)
- [ ] Region click opens detail panel with slide-in from right (300ms)
- [ ] Minimap in corner shows current viewport position
- [ ] Keyboard navigation: arrow keys pan, +/- zoom, Enter selects hovered region

### REQ-48-05: Route Visualization
**Description:** Show animated travel routes between connected regions with directional flow indicators.
**Acceptance Criteria:**
- [ ] Routes rendered as dashed lines with animated dash-offset (marching ants)
- [ ] Route thickness indicates connection strength (trade route vs path)
- [ ] Hovering a route shows tooltip with connected region names
- [ ] Active route (selected in detail panel) highlighted with glow effect
- [ ] Route animations use requestAnimationFrame, not CSS (performance)

## Phase 49: Codex Cards + Relationship Graph

### REQ-49-01: Glassmorphism Entity Cards
**Description:** Redesign Codex entity cards with glassmorphism aesthetic, rich hover states, and category-specific visual treatments.
**Acceptance Criteria:**
- [ ] Cards use frosted glass background (backdrop-filter: blur + semi-transparent bg)
- [ ] Card border has subtle gradient glow matching entity category color
- [ ] Hover lifts card with 3D transform (translateY -4px, subtle shadow increase)
- [ ] Card shows entity portrait/icon, name, type badge, and 2-line description
- [ ] Character cards show faction color stripe on left edge
- [ ] Item cards show rarity indicator (common=gray, rare=blue, epic=purple, legendary=gold)
- [ ] Cards gracefully degrade on browsers without backdrop-filter support

### REQ-49-02: D3 Force-Directed Relationship Graph
**Description:** Interactive force-directed graph showing entity relationships (faction membership, item ownership, region connections) using D3.js.
**Acceptance Criteria:**
- [ ] Graph renders entities as nodes with category-specific shapes (circle=character, diamond=item, hexagon=region)
- [ ] Edges represent relationships with labeled connection type
- [ ] Force simulation settles within 2 seconds with visible node movement
- [ ] Click node to highlight its connections (dim unrelated nodes to 20% opacity)
- [ ] Double-click node navigates to entity detail in Codex
- [ ] Zoom and pan with same controls as world map
- [ ] Graph shows max 100 nodes with pagination/filtering for larger datasets
- [ ] Node size reflects connection count (more connections = larger node)

### REQ-49-03: Entity Detail Flyout
**Description:** Selecting a Codex entity opens a rich detail flyout panel with tabbed sections and cross-reference links.
**Acceptance Criteria:**
- [ ] Flyout slides in from right with 300ms spring animation
- [ ] Header shows entity portrait, name, type, and faction badge
- [ ] Tabs: Overview | Attributes | Relationships | References
- [ ] Overview tab shows AI-generated summary and key stats
- [ ] Attributes tab shows all XML attributes with semantic highlighting (reuses REQ-47-01)
- [ ] Relationships tab shows mini force graph of direct connections
- [ ] References tab lists all files/entities that reference this entity
- [ ] Cross-ref links are clickable and navigate within Codex

### REQ-49-04: Card Grid Layout with Masonry
**Description:** Codex card grid uses masonry layout for visual variety, with smooth reflow on filter changes.
**Acceptance Criteria:**
- [ ] Cards arranged in masonry grid (variable height based on content)
- [ ] Filter changes animate card positions with FLIP technique (300ms)
- [ ] Category filter chips at top with count badges
- [ ] Sort options: Name (A-Z), Type, Relationship count, Recently viewed
- [ ] Empty state shows illustrated placeholder with search suggestion
- [ ] Virtual scrolling for 100+ entities (renders only visible cards)

## Phase 50: Cross-cutting WOW Polish

### REQ-50-01: Page Transition Animations
**Description:** Navigation between major pages (Translator, Game Dev, Codex, Map) uses smooth transitions instead of instant swap.
**Acceptance Criteria:**
- [ ] Page exit animates with fade-out + slight scale-down (200ms)
- [ ] Page enter animates with fade-in + slight scale-up (300ms)
- [ ] Transition direction follows tab order (left-to-right = slide right, right-to-left = slide left)
- [ ] Content skeleton shown during transition if data loading needed
- [ ] Transitions respect prefers-reduced-motion (instant swap fallback)
- [ ] No layout shift during transitions (fixed container dimensions)

### REQ-50-02: Loading State Choreography
**Description:** Replace spinner-only loading states with choreographed skeleton screens and progressive content reveal.
**Acceptance Criteria:**
- [ ] XML file loading shows skeleton tree with pulsing placeholder lines
- [ ] Map loading shows parchment background first, then regions fade in by distance from center
- [ ] Codex loading shows card skeletons with staggered shimmer animation
- [ ] AI summary loading shows typing indicator with partial text reveal
- [ ] All skeletons match actual content dimensions (no layout shift on load)
- [ ] Loading states accessible (aria-busy, screen reader announcements)

### REQ-50-03: Keyboard Shortcuts System
**Description:** Comprehensive keyboard shortcut system with discoverable shortcut hints and a shortcut palette.
**Acceptance Criteria:**
- [ ] Ctrl+K opens command palette with fuzzy search
- [ ] Ctrl+1/2/3/4 switches between main pages
- [ ] Ctrl+F focuses search in current context (XML viewer, Codex, Map)
- [ ] Escape closes any open panel, modal, or palette
- [ ] ? key opens shortcut reference overlay
- [ ] Shortcut hints shown in tooltips and menu items
- [ ] No conflicts with browser or OS shortcuts
- [ ] Shortcuts disabled when input/textarea is focused

### REQ-50-04: Micro-interactions and Feedback
**Description:** Add tactile micro-interactions throughout the UI for a premium feel.
**Acceptance Criteria:**
- [ ] Button press shows subtle scale-down (0.97) with spring-back
- [ ] Toggle switches animate with bounce physics
- [ ] Tab selection has sliding underline indicator with spring animation
- [ ] Copy-to-clipboard shows brief "Copied!" toast with checkmark animation
- [ ] Error states shake horizontally (3 oscillations, 200ms)
- [ ] Success states flash green border pulse (one cycle, 400ms)
- [ ] All micro-interactions use CSS transforms (GPU-accelerated, no layout thrash)

## Out of Scope

| Feature | Reason |
|---------|--------|
| 3D WebGL map rendering | Overkill for showcase — 2D SVG sufficient |
| Real-time multiplayer cursors | Enterprise v4.0 feature |
| AI voice narration of entities | Depends on TTS integration (deferred) |
| Mobile responsive layout | Desktop-first tool — mobile is v4.0 |
| Localization of UI strings | Showcase is English-only for now |
