<objective>
Add fallback reason tracking to image/audio resolution chains and display specific failure reasons in the frontend tabs.

Purpose: MEDIA-01 through MEDIA-04. The backend chains (C1/C3/C7) already resolve LanguageData StringIDs to DDS/WEM files. But when resolution fails, the API returns a generic 404 and the frontend shows "No Image/Audio Context" with no explanation. This plan adds structured fallback reasons so translators see WHY media is missing (entity not found, no texture attribute, file not on disk, no event mapping).

Output: Updated MapDataService with fallback_reason tracking, updated API response models, updated ImageTab/AudioTab with reason display.
</objective>
