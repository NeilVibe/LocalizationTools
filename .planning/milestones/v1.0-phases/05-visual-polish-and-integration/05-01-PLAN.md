    - Test: get_audio_context returns AudioContext for known StrKey with event_name, wem_path, script text
    - Test: get_audio_context returns None for unknown StrKey
    - Test: multi-key index lookup works (StrKey, StringID, KnowledgeKey all resolve same entry)
    - Test: WSL path conversion (F:\perforce\... -> /mnt/f/perforce/...)
    - Test: generate_paths produces correct paths for branch/drive combinations
  </behavior>
  <action>
    1. Create `server/tools/ldm/services/mapdata_service.py`:
       - `MapDataService` class with `_strkey_to_image: Dict[str, ImageContext]` and `_strkey_to_audio: Dict[str, AudioContext]` indexes
       - `ImageContext` dataclass: texture_name (str), dds_path (str), thumbnail_url (str), has_image (bool)
       - `AudioContext` dataclass: event_name (str), wem_path (str), script_kr (str), script_eng (str), duration_seconds (Optional[float])
       - `initialize(branch: str, drive: str)` method that builds indexes from staticinfo XML using MapDataGenerator's LinkageResolver pattern (knowledge lookup -> UITextureName -> DDS path chain)
       - Multi-key indexing: index same entry under StrKey, StringID, and KnowledgeKey for robust lookup
       - `_convert_to_wsl_path(windows_path: str) -> str` helper: `F:\perforce\...` -> `/mnt/f/perforce/...`
       - `KNOWN_BRANCHES` and `PATH_TEMPLATES` adapted from MapDataGenerator/config.py
       - `generate_paths(drive: str, branch: str) -> dict` generating all resolved paths
       - `get_image_context(string_id: str) -> Optional[ImageContext]`
       - `get_audio_context(string_id: str) -> Optional[AudioContext]`
       - Graceful degradation: if paths don't exist, service stays in unloaded state, returns None for all lookups

    2. Create `server/tools/ldm/routes/mapdata.py`:
       - `GET /mapdata/image/{string_id}` -> returns ImageContext JSON or 404
       - `GET /mapdata/audio/{string_id}` -> returns AudioContext JSON or 404
       - `GET /mapdata/context/{string_id}` -> returns both image + audio context combined
       - `POST /mapdata/configure` -> accepts {branch, drive}, re-initializes MapDataService
       - `GET /mapdata/status` -> returns whether service is loaded, which branch/drive, path count
       - All endpoints require auth via `Depends(get_current_active_user_async)` (same pattern as settings.py)
       - Use Pydantic models for response schemas

    3. Register in `server/tools/ldm/router.py`:
       - `from server.tools.ldm.routes.mapdata import router as mapdata_router`
       - `router.include_router(mapdata_router, prefix="/api/ldm")`

    4. Create `tests/unit/ldm/test_mapdata_service.py`:
       - Test MapDataService with mock data (no actual file access)
       - Pre-populate indexes manually, then test get_image_context / get_audio_context lookups
       - Test multi-key resolution (same entry under StrKey and StringID)
       - Test WSL path conversion
       - Test generate_paths with known branch/drive combos
       - Test graceful None returns for unknown keys
  </action>
