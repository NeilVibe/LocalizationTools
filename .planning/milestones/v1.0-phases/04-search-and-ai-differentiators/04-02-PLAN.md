    **Create SemanticResults.svelte** — a dropdown overlay component:
    - Props: `results` (array of {source_text, target_text, similarity, match_type}), `onSelect` (callback with row data), `visible` (boolean)
    - Renders as absolute-positioned dropdown below the search bar
    - Each result row shows: source text (truncated), target text (truncated), similarity percentage badge (use color system: green >= 100%, yellow >= 75%, orange >= 50%)
    - Include search_time_ms display at bottom ("Found N results in Xms")
    - Click handler calls onSelect with the result data
    - Svelte 5 runes: `$state`, `$derived` for filtered/sorted views
    - Close on Escape key or click outside

    **Modify VirtualGrid.svelte** — rewire fuzzy/Similar mode:
    1. Add `let semanticResults = $state([])` and `let semanticSearchTime = $state(0)`
    2. When `searchMode === 'fuzzy'` and user types in search bar, instead of calling loadRows():
       - Fetch `GET /api/ldm/semantic-search?query={searchTerm}&tm_id={activeTmId}&threshold=0.5&max_results=20`
       - Use existing debounce pattern (searchDebounceTimer, 300ms)
       - Store results in semanticResults state
       - If no active TM, show inline message: "Assign a TM to enable semantic search"
    3. Render `<SemanticResults>` overlay when semanticResults has items
    4. On result click: scroll to matching row in grid (by source_text match or row_id if available)
    5. When searchMode changes away from 'fuzzy', clear semanticResults

    **Add AI-suggested badge to grid rows:**
    1. Add `match_source` tracking: in-memory Map/Set tracking which rows had TM applied
    2. When user applies a TM suggestion from TMTab (existing flow), mark that row's match_source
    3. In the grid row rendering, if row has match_source: render a small MachineLearningModel icon (12px) with appropriate color (green=exact, yellow=fuzzy, blue=semantic)
    4. Import MachineLearningModel from carbon-icons-svelte (already used in RightPanel)
    5. Add CSS class `.ai-badge` with subtle styling (no jarring visual, small inline icon)

    **CRITICAL Svelte 5 patterns:**
    - Use `$state()` for all reactive variables
    - Use `$derived()` for computed values
    - Use `$effect()` for side effects
    - Keys in `{#each}` blocks
