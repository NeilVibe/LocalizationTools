# v7.1 Requirements: Security Hardening

## Overview

Full security audit found 28 issues (8 CRITICAL, 7 HIGH, 6 MEDIUM, 7 LOW) across 21 API files and 52 Svelte components. This milestone fixes all CRITICAL and HIGH issues, with MEDIUM/LOW tracked for future work.

## Requirements

### SEC-AUTH: Re-enable Authentication on Disabled Endpoints
- **SEC-AUTH-01**: All 8 admin_telemetry.py endpoints require `require_admin_async`
- **SEC-AUTH-02**: All 6 rankings.py endpoints require `require_admin_async` (currently commented out)
- **SEC-AUTH-03**: logs_async.py `GET /logs/recent` requires `require_admin_async` (currently commented out)
- **SEC-AUTH-04**: merge.py preview + execute endpoints require `get_current_active_user_async`
- **SEC-AUTH-05**: health.py `/api/health/status` upgraded from user auth to `require_admin_async`

### SEC-PATH: Fix Path Traversal Vulnerabilities
- **SEC-PATH-01**: base_tool_api.py `save_uploaded_files` sanitizes filename (use `Path.name` or `secure_filename`)
- **SEC-PATH-02**: updates.py `GET /download/{filename}` validates resolved path stays within UPDATES_DIR
- **SEC-PATH-03**: merge.py validates source_path/target_path/export_path against allowed project directories
- **SEC-PATH-04**: xlstransfer_async.py `POST /test/simple/execute` validates temp_source/temp_dest paths

### SEC-FRONT: Fix Frontend Security Issues
- **SEC-FRONT-01**: ExplorerSearch.svelte HTML-escapes server data before `{@html}` rendering
- **SEC-FRONT-02**: ProjectSettingsModal.svelte uses `getApiBase()` + `getAuthHeaders()` from canonical utility
- **SEC-FRONT-03**: Consolidate 3 remaining local `getAuthHeaders()` duplicates (BranchDrive, ServerStatus, ReferenceSettings)

### SEC-MISC: Fix Remaining HIGH-Severity Issues
- **SEC-MISC-01**: remote_logging.py registration requires shared secret or admin auth
- **SEC-MISC-02**: remote_logging.py IDOR — installation can only query its own status
- **SEC-MISC-03**: remote_logging.py `/installations` list filtered to requesting installation only
- **SEC-MISC-04**: remote_logging.py `/frontend` log injection endpoint requires auth or rate limiting
- **SEC-MISC-05**: base_tool_api.py stderr suppression removed (use proper logging instead)
