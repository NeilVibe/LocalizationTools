# Roadmap: LocaNext v7.1 Security Hardening

## Overview

Fix all CRITICAL and HIGH security issues found in the full-stack audit. 28 total issues (8 CRITICAL, 7 HIGH). This milestone covers auth gaps, path traversal, XSS, IDOR, and frontend security consolidation.

## Phases

- [ ] **Phase 65: Backend Auth Restoration** - Re-enable commented-out auth on telemetry/rankings/logs, add auth to merge/health endpoints
- [ ] **Phase 66: Path Traversal & Input Validation** - Fix path traversal in file upload/download/merge/xlstransfer, add path validation helpers
- [ ] **Phase 67: Frontend Security & Consolidation** - Fix XSS in ExplorerSearch, consolidate getAuthHeaders duplicates, fix missing auth in ProjectSettings
- [ ] **Phase 68: Remote Logging & Misc Security** - Fix IDOR, open registration, log injection, stderr suppression in remote_logging + base_tool_api

## Phase Details

### Phase 65: Backend Auth Restoration
**Goal**: Every API endpoint that exposes user data, system metrics, or performs mutations requires proper authentication — no more "temporarily disabled" auth
**Depends on**: Nothing (first phase)
**Requirements**: SEC-AUTH-01, SEC-AUTH-02, SEC-AUTH-03, SEC-AUTH-04, SEC-AUTH-05
**Success Criteria**:
  1. All 8 admin_telemetry endpoints return 401 when called without valid admin token
  2. All 6 rankings endpoints return 401 when called without valid admin token
  3. logs_async GET /logs/recent returns 401 without admin auth
  4. merge.py preview/execute return 401 without user auth
  5. health.py /api/health/status returns 401 without admin auth
**Estimated Plans**: 1 (single sweep — all files are auth additions only)

### Phase 66: Path Traversal & Input Validation
**Goal**: No user-supplied path can escape its intended directory — all file operations validated against allowed roots
**Depends on**: Nothing (independent of Phase 65)
**Requirements**: SEC-PATH-01, SEC-PATH-02, SEC-PATH-03, SEC-PATH-04
**Success Criteria**:
  1. base_tool_api `save_uploaded_files` strips directory components from filenames
  2. updates.py download validates resolved path is within UPDATES_DIR
  3. merge.py validates all paths are within project-configured directories
  4. xlstransfer test endpoint validates paths are within temp directory
**Estimated Plans**: 1-2 (path validation utility + apply to all endpoints)

### Phase 67: Frontend Security & Consolidation
**Goal**: No XSS vectors in Svelte components, all fetch calls use canonical auth utility
**Depends on**: Nothing (independent)
**Requirements**: SEC-FRONT-01, SEC-FRONT-02, SEC-FRONT-03
**Success Criteria**:
  1. ExplorerSearch `{@html}` content is HTML-escaped before rendering
  2. ProjectSettingsModal uses getApiBase() + getAuthHeaders() from $lib/utils/api.js
  3. Zero local getAuthHeaders() duplicates remain (all consolidated to api.js import)
**Estimated Plans**: 1

### Phase 68: Remote Logging & Misc Security
**Goal**: Remote logging endpoints can't be abused for data enumeration, injection, or unauthorized access
**Depends on**: Nothing (independent)
**Requirements**: SEC-MISC-01, SEC-MISC-02, SEC-MISC-03, SEC-MISC-04, SEC-MISC-05
**Success Criteria**:
  1. Installation registration requires a shared secret header
  2. GET /status/{id} validates installation_id matches the authenticated installation
  3. GET /installations returns only the requesting installation's data
  4. POST /frontend requires valid API key
  5. base_tool_api background tasks log stderr properly instead of suppressing
**Estimated Plans**: 1-2
