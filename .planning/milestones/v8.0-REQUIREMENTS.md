# v8.0 Requirements: Service Layer Extraction

## Overview

Extract business logic from 6 thick API modules (4,032 lines) into proper service classes. Route handlers become thin HTTP dispatchers. Enables unit testing, reduces duplication, and creates seams for future refactoring.

## Requirements

### SVC-STATS: Stats & Rankings Service
- **SVC-STATS-01**: StatsService extracts all 14 stats.py query builders + aggregation logic
- **SVC-STATS-02**: RankingsService extracts all 6 rankings.py query builders
- **SVC-STATS-03**: Shared `_get_date_range(period)` helper eliminates date-range duplication
- **SVC-STATS-04**: N+1 queries replaced with JOINs/window functions

### SVC-AUTH: Auth Service
- **SVC-AUTH-01**: AuthService encapsulates login (rate limiting + verify + token + audit)
- **SVC-AUTH-02**: AuthService encapsulates register + admin CRUD + password change
- **SVC-AUTH-03**: Route handlers are ≤10 lines (parse request → call service → format response)

### SVC-TELEMETRY: Telemetry & Remote Logging Services
- **SVC-TELEM-01**: TelemetryService extracts admin_telemetry.py query logic
- **SVC-TELEM-02**: RemoteLoggingService extracts log ingestion, session tracking, summary upsert
- **SVC-TELEM-03**: N+1 queries in installations list fixed with JOINs

### SVC-INFRA: Infrastructure Services
- **SVC-INFRA-01**: DbHealthService extracts admin_db_stats.py raw SQL + health assessment
- **SVC-INFRA-02**: HealthService consolidates health.py helper functions
- **SVC-INFRA-03**: ProgressService separates WebSocket dispatch from CRUD in progress_operations.py
