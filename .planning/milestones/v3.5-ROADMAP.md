## Phase 47: XML Viewer WOW Polish

**Goal:** Transform the XML viewer from functional data display to a visually intelligent interface where attribute types are instantly recognizable through semantic coloring, hover interactions reveal context, and tree navigation feels fluid.

**Effort:** ~4 hours
**Dependencies:** v3.4 complete (GameData Tree + Context Panel)
**Success Criteria:** Every attribute click/hover produces a delightful visual response. A new user can instantly distinguish cross-refs from editable fields from identity keys by color alone.

**Requirements:** 4
- **REQ-47-01:** Smart Attribute Highlighting — semantic colors by attribute type (cross-ref=blue, editable=green, identity=gold, media=purple, numeric=cyan)
- **REQ-47-02:** Hover Tooltips with Preview — contextual tooltip showing target entity, thumbnail, or edit hint after 300ms delay
- **REQ-47-03:** Smooth Expand/Collapse Animations — height transitions with staggered child reveal and rotating chevron
- **REQ-47-04:** Inline Attribute Search — Ctrl+F within viewer with yellow highlights, match navigation, and count display

**Plans (to be created):**
1. **Plan 47-A:** Attribute classification engine + semantic color system
2. **Plan 47-B:** Tooltip component + cross-ref preview resolution
3. **Plan 47-C:** Tree animation system + inline search overlay

**Key Components:**
- `locaNext/src/lib/components/ldm/GameDataTree.svelte` — tree rendering
- `locaNext/src/lib/components/ldm/GameDataContextPanel.svelte` — context display
- New: `AttributeHighlighter.svelte`, `AttributeTooltip.svelte`

## Execution Strategy

### Recommended Order
1. **Phase 47 first** — establishes animation patterns and attribute classification reused everywhere
2. **Phase 48 + 49 in parallel** — independent pages, can use agent teams (1 teammate per phase)
3. **Phase 50 last** — cross-cutting polish requires all pages to be in final state

### Swarm Configuration
- **Phase 47:** Single session (3 plans, tightly coupled)
- **Phase 48 + 49:** Agent team with 2 teammates (independent workstreams)
- **Phase 50:** Single session (cross-cutting, touches all pages)

### Testing Approach
- **API tests first** for any new backend endpoints (gamedata cross-ref resolution)
- **Playwright screenshots** for every visual requirement (REQ-47-01 colors, REQ-48-01 parchment, etc.)
- **Performance budget:** No phase should add >50ms to page load, no animation should drop below 60fps
- **Accessibility:** All animations respect prefers-reduced-motion, all interactions keyboard-accessible

### Risk Mitigation
| Risk | Mitigation |
|------|------------|
| D3 bundle size | Tree-shake: import only d3-force, d3-zoom, d3-selection (~30KB) |
| backdrop-filter browser support | Graceful degradation to solid semi-transparent background |
| Animation performance on large datasets | Virtual scrolling + skip stagger for 50+ items |
| rough.js dependency | Optional enhancement — CSS fallback for hand-drawn borders |
