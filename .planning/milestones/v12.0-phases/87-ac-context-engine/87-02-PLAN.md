    **Part A: Add POST /tm/context endpoint to tm_search.py**

    Add a new endpoint at the bottom of `server/tools/ldm/routes/tm_search.py`:

    ```python
    from pydantic import BaseModel
    from server.tools.ldm.indexing.context_searcher import ContextSearcher
    from server.tools.ldm.indexing.indexer import TMIndexer

    class ContextSearchRequest(BaseModel):
        source: str
        tm_id: int
        max_results: int = 10

    @router.post("/tm/context")
    async def context_search(
        request: ContextSearchRequest,
        tm_repo: TMRepository = Depends(get_tm_repository),
        current_user: dict = Depends(get_current_active_user_async)
    ):
    ```

    Endpoint logic:
    1. Validate tm_id exists via `tm_repo.get(request.tm_id)` — 404 if not found
    2. Get TMIndexer instance, call `load_indexes(request.tm_id)` to get indexes dict
       - NOTE: In production, indexes are cached in memory (the LDM service manages this). For now, load fresh. Phase 88 will optimize caching.
    3. Create `ContextSearcher(indexes)` and call `searcher.search(request.source, request.max_results)`
    4. Return the result dict directly (it already has the right shape)
    5. Wrap in try/except, log errors with `[TM-SEARCH] [CONTEXT]` prefix per project convention
    6. If indexes not found (FileNotFoundError), return 404 with "TM indexes not built"

    **Part B: Add performance benchmark to test_context_searcher.py**

    Add a new test class `TestContextSearcherPerformance` to the existing test file:

    1. Helper: `_build_korean_tm_entries(count: int)` — generates synthetic Korean TM data:
       - Use Korean syllable range (0xAC00-0xD7AF) to generate random Korean strings
       - Each entry: {"id": i, "source_text": random_korean(20-80 chars), "target_text": random_korean(20-80 chars)}
       - Normalize keys with `normalize_for_hash()`
       - Build whole_lookup and line_lookup dicts from these entries

    2. Helper: `_build_indexes_with_ac(whole_lookup, line_lookup)` — creates indexes dict with AC automatons:
       - Build whole_automaton from whole_lookup keys
       - Build line_automaton from line_lookup keys
       - Return complete indexes dict

    3. Test `test_performance_1000_entries`:
       - Generate 1000 Korean TM entries
       - Build indexes with AC
       - Pick 10 random source texts from the entries (partial substrings to trigger all tiers)
       - Time each `searcher.search()` call
       - Assert each call < 100ms
       - Assert average < 50ms (headroom)

    4. Test `test_performance_2000_entries_consecutive`:
       - Generate 2000 entries
       - Run 10 consecutive searches
       - Assert ALL < 100ms each

    5. If n={2,3,4,5} Jaccard against all 1000+ entries is too slow (> 100ms), the ContextSearcher should limit Tier 3 candidates:
       - Only compute Jaccard for entries where at least ONE 2-gram from source appears in the entry (pre-filter)
       - This is an optimization hint for the executor — implement in context_searcher.py if perf test fails on first run
