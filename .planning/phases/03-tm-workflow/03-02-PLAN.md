    1. Create `locaNext/src/lib/utils/wordDiff.js`:
       - Export `tokenize(text)` -- splits on whitespace AND between CJK characters using regex: `[\u3000-\u9fff\uac00-\ud7af]|[^\s\u3000-\u9fff\uac00-\ud7af]+`
       - Export `computeWordDiff(original, match)` -- uses Longest Common Subsequence (LCS) algorithm to find unchanged tokens, marks added/removed tokens
       - Returns `{ original: [{text, type}], match: [{text, type}] }` where type is 'same', 'added', or 'removed'
       - Handle edge cases: empty strings, identical strings (all 'same'), completely different strings
       - Korean syllables (U+AC00-U+D7AF) are each treated as individual tokens for granular diff

    2. Create `locaNext/src/lib/components/ldm/TMTab.svelte`:
       - Props (Svelte 5 runes): `let { selectedRow = $bindable(null), tmMatches = $bindable([]), tmLoading = $bindable(false) } = $props()`
       - `getMatchColor(score)` function returning color+label:
         - score >= 1.0: { color: '#24a148', label: 'Exact', type: 'green' }
         - score >= 0.92: { color: '#c6a300', label: 'High Fuzzy', type: 'yellow' }
         - score >= 0.75: { color: '#ff832b', label: 'Fuzzy', type: 'orange' }
         - below 0.75: { color: '#da1e28', label: 'Low', type: 'red' }
       - Each match displays: color-coded percentage badge (inline style or CSS class), source text, target text, match metadata (TM name, type)

       - Loading state: skeleton/spinner when tmLoading is true
       - Empty state: "Select a segment to see TM matches" or "No matches found"
       - Dispatch 'applyTM' event when user clicks/tabs a match (preserve existing keyboard apply behavior from TMQAPanel)
       - CSS: Clean, professional styling. Percentage badge as small pill with background color. Diff tokens with subtle highlight colors. Compact layout for multiple matches.
