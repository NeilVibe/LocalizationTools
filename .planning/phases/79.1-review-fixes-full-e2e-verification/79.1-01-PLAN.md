**FIX-01: Merge route conflict.**
Both `files.py` line 926 and `merge.py` line 56 register `POST /files/{file_id}/merge`. Since `files_router` is included before `merge_router` in router.py (line 81 vs 103), files.py always wins. The files.py merge is an OLDER "export-style" merge (accepts multipart UploadFile, merges reviewed rows back into original file). The merge.py endpoint is the TranslatorMergeService merge (accepts JSON body with source_file_id + match_mode).

**Fix:** Rename the files.py route from `/files/{file_id}/merge` to `/files/{file_id}/export-merge` at line 926. This is the correct semantic — it merges reviewed edits back into the original file for download/export. Update the docstring to clarify it is the "export merge" endpoint. Do NOT change merge.py — it is the correct TranslatorMergeService endpoint.

After fixing, also update `tests/e2e/test_merge_qa_pipeline.py`:
- Remove the `XFAIL_REASON` constant (lines 94-98)
- Remove all `@pytest.mark.xfail(reason=self.XFAIL_REASON)` decorators from test methods in the `TestTranslatorMerge` class
- The tests should now be real tests (pass or fail), not xfail

**FIX-02: test_langdata_media.py fixture duplication.**
Remove lines 31-59 from `tests/e2e/test_langdata_media.py` — the local `client()` and `auth_headers()` fixture definitions. These shadow the session-scoped fixtures in `tests/e2e/conftest.py`. Also remove the `from fastapi.testclient import TestClient` and `from server.main import app` imports (lines 20-21) since they are only used by the removed fixtures.
