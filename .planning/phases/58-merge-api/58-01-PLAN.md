Create server/api/merge.py with:

1. **Pydantic models (co-located, following settings.py pattern):**
   - `MergePreviewRequest(BaseModel)`: fields `source_path: str`, `target_path: str`, `export_path: str`, `match_mode: str = "strict"`, `only_untranslated: bool = False`, `stringid_all_categories: bool = False`, `multi_language: bool = False`
   - `MergePreviewResponse(BaseModel)`: fields `files_processed: int = 0`, `total_corrections: int = 0`, `total_matched: int = 0`, `total_updated: int = 0`, `total_not_found: int = 0`, `total_skipped: int = 0`, `total_skipped_translated: int = 0`, `overwrite_warnings: list[str] = []`, `errors: list[str] = []`, `per_language: dict | None = None`, `scan: dict | None = None`
   - `MergeExecuteRequest(BaseModel)`: same fields as MergePreviewRequest (Plan 02 will use this)

2. **Module-level merge guard:**
   - `_merge_in_progress: bool = False` flag
   - Used by execute endpoint (Plan 02) to return 409 if merge already running

3. **Router:**
   - `router = APIRouter(prefix="/api/merge", tags=["Merge"])`

4. **POST /preview endpoint:**
   - Import `translate_wsl_path` from `server.api.settings`
   - Import `execute_transfer`, `execute_multi_language_transfer`, `scan_source_languages`, `MATCH_MODES` from `server.services.transfer_adapter`
   - Validate `body.match_mode in MATCH_MODES`, return 422 with clear error if invalid
   - Translate all 3 paths via `translate_wsl_path()`
   - If `body.multi_language is True`:
     - Call `execute_multi_language_transfer(source_path=..., target_path=..., export_path=..., match_mode=..., only_untranslated=..., stringid_all_categories=..., dry_run=True)` via `asyncio.to_thread()`
     - Map result to MergePreviewResponse: `total_matched`, `total_skipped`, `per_language`, `scan` from result
   - If `body.multi_language is False`:
     - Call `execute_transfer(source_path=..., target_path=..., export_path=..., match_mode=..., only_untranslated=..., stringid_all_categories=..., dry_run=True)` via `asyncio.to_thread()`
     - Map result to MergePreviewResponse using result dict keys: `files_processed`, `total_matched`, `total_updated`, `total_not_found`, `total_skipped`, `total_skipped_translated`
     - Extract overwrite warnings: iterate `result.get("file_results", {})`, for each file with `matched > 0` AND `not only_untranslated`, add warning string `"{filename}: {matched} entries will be overwritten"`
   - Wrap entire logic in try/except, return MergePreviewResponse with `errors=[str(exc)]` on failure
   - Use `from loguru import logger` for all logging (NEVER print())
   - Use `from __future__ import annotations` at top of file

5. **Register in main.py:**
   - Add after the settings_api registration (line 451), before the LDM router (line 456):
   ```python
   # Include Merge API (Phase 58: Merge preview + execute)
   from server.api import merge as merge_api
   app.include_router(merge_api.router)
   ```

- NEVER run QT functions directly in async handler -- always `asyncio.to_thread()`
- NEVER forget `translate_wsl_path()` on ALL three paths
- NEVER serialize Path objects without `default=str` in json.dumps
- Use `from __future__ import annotations` (Python compat rule from CLAUDE.md)
