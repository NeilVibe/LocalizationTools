**Schema file** `server/tools/ldm/schemas/codex_audio.py`:

Create Pydantic v2 models following the codex_characters.py pattern:

1. `AudioCardResponse(BaseModel)` -- card/list entry for grid view:
   - `event_name: str` -- the audio event name (D10 key)
   - `string_id: Optional[str]` -- linked StringId from D11
   - `script_kr: Optional[str]` -- Korean script line from C4
   - `script_eng: Optional[str]` -- English script line from C5
   - `export_path: Optional[str]` -- category from D20 (e.g. "Dialog/QuestDialog")
   - `has_wem: bool` -- whether WEM file exists on disk
   - `xml_order: Optional[int]` -- ordering from D21

2. `AudioDetailResponse(BaseModel)` -- full detail for single audio entry:
   - All fields from AudioCardResponse
   - `wem_path: Optional[str]` -- full WEM file path (string)
   - `stream_url: Optional[str]` -- computed URL for WAV streaming

3. `AudioListResponse(BaseModel)` -- paginated list:
   - `items: List[AudioCardResponse]`
   - `total: int`
   - `offset: int`
   - `limit: int`
   - `has_more: bool`
   - `category_filter: Optional[str]`

4. `AudioCategoryNode(BaseModel)` -- tree node for category sidebar:
   - `name: str` -- directory name (e.g. "QuestDialog")
   - `full_path: str` -- full relative path (e.g. "Dialog/QuestDialog")
   - `count: int` -- number of audio entries in this category
   - `children: List[AudioCategoryNode]` -- nested subcategories

5. `AudioCategoryTreeResponse(BaseModel)` -- tree response:
   - `categories: List[AudioCategoryNode]`
   - `total_events: int`

**Routes file** `server/tools/ldm/routes/codex_audio.py`:

Create 4 endpoints following the codex_items.py pattern:

1. `GET /codex/audio/categories` -- Category tree from D20:
   - Call `mega = get_mega_index()`
   - Group `mega.event_to_export_path` by directory parts
   - Build nested `AudioCategoryNode` tree
   - Return `AudioCategoryTreeResponse`

2. `GET /codex/audio/stream/{event_name}` -- WEM-to-WAV streaming:
   - Look up WEM path: `mega.get_audio_path_by_event(event_name)`
   - If WEM path is None or doesn't exist, return 404
   - Convert using existing pattern: `convert_to_wsl_path()` + `await asyncio.to_thread(converter.convert_wem_to_wav, Path(wsl_path))`
   - Return `FileResponse(str(wav_path), media_type="audio/wav")`
   - Import `convert_to_wsl_path` from `server.tools.ldm.services.perforce_path_service`
   - Import `get_media_converter` from `server.tools.ldm.services.media_converter`

3. `GET /codex/audio/{event_name}` -- Full detail:
   - Build `AudioDetailResponse` with all MegaIndex lookups
   - Include `stream_url` computed as `/api/ldm/codex/audio/stream/{event_name}`
   - Return 404 if event not found in D11

4. `GET /codex/audio/` -- Paginated list with filtering:
   - Query params: `offset`, `limit`, `category` (D20 prefix filter), `q` (text search)
   - Iterate `mega.event_to_stringid` (D11) as the master list of audio events (these are events that have StringId linkage -- the interesting ones)
   - For category filter: match `mega.event_to_export_path[event].startswith(category)`
   - For text search `q`: search across event_name, script_kr (C4), script_eng (C5), string_id (D11)
   - Sort by xml_order (D21) if available, else alphabetical by event_name
   - Build `AudioCardResponse` for each entry
   - has_wem: `mega.get_audio_path_by_event(event) is not None`

Use `APIRouter(prefix="/codex/audio", tags=["Codex Audio"])`.
Use `Depends(get_current_active_user_async)` on all endpoints except stream (matching mapdata pattern where stream is unauthenticated for `<audio>` element compatibility).
