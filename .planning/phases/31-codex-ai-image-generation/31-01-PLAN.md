       ```python
       @router.get("/image-gen/status")
       async def image_gen_status(current_user = Depends(get_current_active_user_async)):
           svc = get_ai_image_service()
           return {"available": svc.available}
       ```

    2. **POST /codex/generate-image/{entity_type}/{strkey}** -- Generate a single AI image:
       - Import `asyncio` and `get_ai_image_service` from the new service
       - Get entity via `_get_codex_service().get_entity(entity_type, strkey)` -- 404 if not found
       - Check `svc.available` -- return 503 with `{"detail": "AI image generation unavailable (GEMINI_API_KEY not set)"}` if false
       - Check if already cached: `svc.get_cached_image_path(strkey)` -- if exists and no `force` query param, return existing URL
       - Call `png_bytes = await asyncio.to_thread(svc.generate_image, entity)` -- wrap in try/except for Gemini errors
       - On success: `svc.save_to_cache(strkey, png_bytes, prompt)` and return `{"status": "generated", "image_url": f"/api/ldm/codex/image/{strkey}"}`
       - On failure: return 500 with error detail, log with loguru
       - Accept optional `force: bool = Query(False)` to regenerate even if cached

    3. **GET /codex/image/{strkey}** -- Serve a cached AI-generated image:
       - Import `Response` from `fastapi`
       - Get cached path via `svc.get_cached_image_path(strkey)` -- 404 if not found
       - Read bytes and return `Response(content=png_bytes, media_type="image/png", headers={"Cache-Control": "public, max-age=604800"})`
