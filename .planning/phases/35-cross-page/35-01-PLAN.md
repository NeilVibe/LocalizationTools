# Phase 35 Plan 01: Cross-Page Consistency Enforcement

## Objective

Enforce visual and behavioral consistency across all 5 pages so the app feels like ONE product. Replace custom headers with shared PageHeader, ensure dark mode tokens everywhere, standardize sidebar navigation, and ensure ErrorState is used for all error recovery.

## Context

- @locaNext/src/lib/components/common/PageHeader.svelte -- shared header component
- @locaNext/src/lib/components/common/ErrorState.svelte -- shared error component
- @locaNext/src/lib/components/pages/*.svelte -- 5 page components
- @locaNext/src/routes/+layout.svelte -- sidebar navigation

## Tasks

### Task 1: Replace custom headers with shared PageHeader (XPG-01, XPG-04)
type="auto"

Replace the custom header sections in all 5 pages with the shared PageHeader component. Also ensure ErrorState is used consistently for error recovery on all pages.

**Pages to update:**
1. CodexPage.svelte -- has custom `.codex-header` div, replace with PageHeader; replace `.codex-error` with ErrorState
2. WorldMapPage.svelte -- has custom `.worldmap-header` div, replace with PageHeader (already uses ErrorState)
3. TMPage.svelte -- has custom `.tm-page-header` div with h2, replace with PageHeader preserving Upload/Settings action buttons
4. GameDevPage.svelte -- has custom `.explorer-header` in left panel; this is a panel header not page header, keep as-is (no full-width page header needed for split-panel layout)
5. GridPage.svelte -- has custom `.grid-toolbar` which is a contextual toolbar (back button + file info), keep as-is (not a page header)

**Done when:**
- CodexPage, WorldMapPage, TMPage use PageHeader component
- CodexPage error state uses ErrorState with retry
- All PageHeaders render with consistent height/padding/typography
- No custom header CSS duplicating PageHeader styles

### Task 2: Sidebar consistency + dark mode audit + error handling (XPG-02, XPG-03)
type="auto"

Audit sidebar navigation in +layout.svelte for consistent active/hover states. Scan all pages for hardcoded colors. Ensure all pages have proper error recovery.

**Done when:**
- Sidebar nav tabs have consistent focus/active/hover styles using design tokens
- No hardcoded hex colors remain in page components (except intentional TYPE_COLORS/NODE_COLORS documented as design decisions)
- All pages that fetch data have ErrorState with retry capability

## Verification

- All 5 pages render without errors
- Navigating between pages shows consistent header styling
- No hardcoded colors in page components outside of documented exceptions

## Success Criteria

1. 3 pages (Codex, WorldMap, TM) use PageHeader component
2. 2 pages (GameDev, Grid) retain specialized toolbars (documented as intentional)
3. All pages use design tokens for colors
4. ErrorState with retry used on Codex and WorldMap error states
