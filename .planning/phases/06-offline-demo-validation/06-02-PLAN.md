  <action>
Create unit tests for the 3-mode detection logic in factory.py. These are pure logic tests -- no live server needed.

Tests to write:

1. **_is_offline_mode with OFFLINE_MODE_ header**: Create a mock Request with `Authorization: Bearer OFFLINE_MODE_abc123`, verify returns True.
2. **_is_offline_mode with regular JWT**: Create a mock Request with `Authorization: Bearer eyJhbG...`, verify returns False.
3. **_is_offline_mode with no header**: Create a mock Request with no Authorization header, verify returns False.
4. **_is_server_local when sqlite**: Patch `server.config.ACTIVE_DATABASE_TYPE = "sqlite"`, verify returns True.
5. **_is_server_local when postgresql**: Patch `server.config.ACTIVE_DATABASE_TYPE = "postgresql"`, verify returns False.
6. **Factory function routing (server_local)**: Patch config to "sqlite", call `get_file_repository` with a non-offline request, verify returns SQLiteFileRepository with SchemaMode.SERVER.
7. **Factory function routing (offline)**: Call `get_file_repository` with OFFLINE_MODE_ header, verify returns SQLiteFileRepository with SchemaMode.OFFLINE.
8. **Factory function routing (all 9 repos)**: Parametrize across all 9 `get_*_repository` functions, verify each returns SQLite variant when in server_local mode.

Use `unittest.mock.MagicMock` for Request objects. Use `unittest.mock.patch` for config. The factory functions need `db` and `current_user` params -- mock those too (they're unused in SQLite path).

Key: These tests prove OFFL-03 (transparent mode switching) at the factory level -- the routing logic that ALL routes depend on.
