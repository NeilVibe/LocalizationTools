    2. Fix the race condition in VirtualGrid.svelte `confirmInlineEdit()`:
       - The bug: `confirmInlineEdit()` does NOT set `isCancellingEdit = true` before clearing `inlineEditingRowId`. When blur fires after confirm, `saveInlineEdit()` checks `if (!inlineEditingRowId || isCancellingEdit)` — since `inlineEditingRowId` is already null, it returns early. BUT if timing allows blur to fire BEFORE `inlineEditingRowId` is cleared, a double-save occurs.
       - Fix: Set `isCancellingEdit = true` at the START of `confirmInlineEdit()` (before the async save), then reset it after the move-to-next-row completes (via setTimeout(0) like cancelInlineEdit does).
       - Also add a `isConfirming` guard flag to prevent re-entry during the async save operation.

    3. Verify the `handleInlineEditKeydown` handler for Ctrl+S calls `confirmInlineEdit()` and NOT `saveInlineEdit()` directly.

    IMPORTANT: Do NOT rewrite VirtualGrid from scratch. Make surgical edits only.
    IMPORTANT: Use Svelte 5 runes syntax only ($state, $derived, $effect). No $: reactive statements.
  </action>

  </verify>
