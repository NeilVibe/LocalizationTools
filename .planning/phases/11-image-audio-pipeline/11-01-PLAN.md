  <action>
    Create `server/tools/ldm/services/media_converter.py` with a `MediaConverter` class and `get_media_converter()` singleton accessor. Port the conversion logic from the research document (Pattern 1):

    **DDS-to-PNG (`convert_dds_to_png`):**
    - Accept `dds_path: Path` and `max_size: int = 256`
    - Return `Optional[bytes]` (PNG bytes or None)
    - Use `PIL.Image.open(dds_path)`, convert to RGBA, thumbnail to max_size, save to BytesIO as PNG
    - In-memory LRU cache keyed by `str(dds_path)`, default 100 entries
    - Wrap in try/except, log errors with loguru, return None on failure

    **WEM-to-WAV (`convert_wem_to_wav`):**
    - Accept `wem_path: Path`, return `Optional[Path]` (WAV path or None)
    - Use `hashlib.md5(str(wem_path).encode()).hexdigest()[:8]` for cache key
    - Store converted WAVs in `tempfile.gettempdir() / "locanext_audio"`
    - Check cache: if WAV exists and mtime >= WEM mtime, return cached
    - Find vgmstream-cli via `_find_vgmstream()`: check `shutil.which("vgmstream-cli")`, then check `server/tools/ldm/bin/vgmstream-cli`
    - Run `subprocess.run([vgmstream_path, "-o", wav_path, wem_path], capture_output=True, text=True, timeout=60)`
    - Return wav_path on success, None on failure

    **Singleton:**
    - `get_media_converter()` pattern matching `get_mapdata_service()`
    - Clear caches method `clear_caches()` for re-initialization

    Use `from __future__ import annotations` at top. Use loguru logger for all logging.

    **Tests:** Create `tests/unit/ldm/test_media_converter.py` with three test classes (TestDdsConversion, TestWemConversion, TestGracefulFallback). Use the existing DDS fixture at `tests/fixtures/mock_gamedata/textures/character_kira.dds` for real DDS tests. Mock subprocess for WEM tests. Use `tmp_path` fixture for temp directories.
