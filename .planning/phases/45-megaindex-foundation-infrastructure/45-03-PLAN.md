   **Phase 1 - Foundation (parallel-safe, no deps):**
   - D9: `_scan_dds_textures(texture_folder)` -- `rglob("*.dds")`, key=`stem.lower()`, value=Path

   **Phase 2 - Entity Parse (depends on D1):**
   - D2: `_parse_character_info(character_folder)` -- parse `characterinfo_*.staticinfo.xml`, create CharacterEntry. Resolve KnowledgeKey against D1 for desc/name enrichment.
   - D3+D14: `_parse_item_info(item_folder, knowledge_folder)` -- parse `<ItemInfo>` from `iteminfo/` AND `<ItemGroupInfo>` for hierarchy (D14). Create ItemEntry with inspect_entries from nested `<InspectData>/<PageData>`.

   - D8: `_parse_gimmick_info(staticinfo_folder)` -- parse `gimmickinfo_*.staticinfo.xml`.

   **Phase 3 - Localization (no entity deps):**
   - D12: `_parse_loc_strorigin(loc_folder)` -- parse `languagedata_KOR.xml`, `<LocStr>` elements: StringId/StringID attr -> StrOrigin attr.

   - D11+D20+D21: `_parse_export_events(export_folder)` -- parse export__ XMLs for SoundEventName/EventName + StringId. Build D11 (event_to_stringid, key=event.lower()), D20 (event_to_export_path), D21 (event_to_xml_order with global counter).
   - D17+D18: `_parse_export_loc(export_folder)` -- parse export__/*.loc.xml for StringId sets per file (D17) and ordered StringId lists per file (D18).

   **Phase 4 - VRS (optional, depends D11+D21):**
   - Skip for now (VRS is xlsx-based, optional ordering). Log `[MEGAINDEX] VRS ordering skipped (future enhancement)`.

   **Phase 5 - Broad scan:**
   - D19: `_scan_devmemo(staticinfo_folder)` -- broad `rglob("*.xml")` scan of StaticInfo folder. For each element with StrKey attr, extract DevMemo or DevComment attr. Key=strkey.lower().

   **Phase 6 - Reverse dicts (depends on Phases 1-3):**
   - R1: `_build_name_kr_to_strkeys()` -- invert name fields from D1, D2, D3, D4. Key=korean_name, value=List[(entity_type, strkey)].
   - R2: `_build_knowledge_key_to_entities()` -- scan D2, D3, D4, D7 knowledge_key fields.
   - R3: `_build_stringid_to_event()` -- invert D11.

   - R7: `_build_group_key_to_items()` -- from D3 group scanning.

   **Phase 7 - Composed dicts (depends on Phases 1-6):**

   - C2: `_build_strkey_to_audio_path()` -- entities with knowledge_key, chain to event, chain to WEM.
   - C3: `_build_stringid_to_audio_path()` -- R3 + D10.
   - C4: `_build_event_to_script_kr()` -- D11 + D12.

   - C6: `_build_entity_strkey_to_stringids()` -- entity.source_file -> D17 + Korean text matching via R6. Follow the bridge mechanism from MEGAINDEX_DESIGN.md Section 6D.
   - C7: `_build_stringid_to_entity()` -- invert C6.
