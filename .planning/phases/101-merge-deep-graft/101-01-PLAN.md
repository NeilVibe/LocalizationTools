# Plan 01: Core Merge Fix — Identical-Skip + Detailed Results

**Goal:** Fix the fundamental merge algorithm so it SKIPS identical rows instead of blindly overwriting, and returns detailed per-row results.

**Root cause of BUG-21 + BUG-13:** The merge matched 169,650 rows and "updated" all of them in DB — but the existing target values were ALREADY CORRECT. So nothing visibly changed. The merge has NO `if new != old: update` check.

## Tasks

### T1: Add identical-skip check to `_merge_single_mode` and `_merge_cascade`

**File:** `server/tools/ldm/services/translator_merge.py`

Current code (both _merge_single_mode and _merge_cascade):
```python
if matched_text is not None:
    updated = dict(target)
    updated["target"] = matched_text
    result.updated_rows.append(updated)
    result.matched += 1
```

Fix — add identical check (graft from QT xml_transfer.py L574):
```python
if matched_text is not None:
    result.matched += 1
    result.match_type_counts[mode] += 1
    existing_target = (target.get("target") or "").strip()
    if matched_text.strip() == existing_target:
        # IDENTICAL — skip write, count as unchanged
        result.unchanged += 1
        result.details.append({
            "row_id": target.get("id"),
            "string_id": target.get("string_id", ""),
            "status": "UNCHANGED",
        })
    else:
        # DIFFERENT — queue for DB write
        updated = dict(target)
        updated["target"] = matched_text
        result.updated_rows.append(updated)
        result.details.append({
            "row_id": target.get("id"),
            "string_id": target.get("string_id", ""),
            "status": "UPDATED",
            "old": existing_target[:80],
            "new": matched_text[:80],
        })
```

Apply this pattern in:
- `_merge_single_mode` (line ~456)
- `_merge_cascade` — strict block (~484), stringid block (~494), strorigin block (~506), descorigin block (~520), filename block (~530), fuzzy block (~546)

### T2: Extend MergeResult dataclass

```python
@dataclass
class MergeResult:
    matched: int = 0          # Total rows that found a correction match
    updated: int = 0          # Actually changed (new != old)  ← NEW
    unchanged: int = 0        # Matched but identical ← NEW
    skipped: int = 0          # Source rows filtered by skip guards
    total: int = 0
    match_type_counts: dict = field(default_factory=...)
    updated_rows: list[dict] = field(default_factory=list)
    details: list[dict] = field(default_factory=list)  ← NEW
```

## Verification

- Merge with source that has IDENTICAL translations → `updated: 0`, `unchanged: N`
- Merge with source that has DIFFERENT translations → `updated: N`, `unchanged: 0`
- Mix: some identical, some different → correct split
- DB only gets writes for UPDATED rows (not unchanged)
