# Plan 02: Merge Options Wiring + Progress Feedback

**Goal:** Wire the 6 frontend merge options that are currently silently dropped by the backend, and add real-time progress feedback during merge.

## Tasks

### T1: Extend MergeRequest Pydantic model (BUG-16)

**File:** `server/tools/ldm/routes/merge.py`

Add fields to MergeRequest:
```python
class MergeRequest(BaseModel):
    source_file_id: int
    match_mode: str = "cascade"
    threshold: float = 0.85
    is_cjk: bool = False
    # NEW — options currently silently dropped:
    transfer_scope: str = Field(default="all", description="'all' or 'untranslated'")
    ignore_spaces: bool = Field(default=False, description="Whitespace-insensitive matching")
    ignore_punctuation: bool = Field(default=False, description="Ignore sentence-ending punctuation")
    all_categories: bool = Field(default=False, description="StringID mode: include ALL categories")
    non_script_only: bool = Field(default=False, description="Strict mode: exclude SCRIPT categories")
    unique_only: bool = Field(default=False, description="StrOrigin mode: skip duplicate source texts")
```

### T2: Wire options into TranslatorMergeService

**File:** `server/tools/ldm/services/translator_merge.py`

Add to `merge_files()` signature:
```python
def merge_files(self, ..., transfer_scope="all", ignore_spaces=False, ignore_punctuation=False, ...):
```

Wire `transfer_scope="untranslated"`:
- In the match+update block, BEFORE updating: check if existing target is already translated (non-empty, non-Korean)
- If `transfer_scope == "untranslated"` and existing target is already translated → skip, count as `skipped_translated`

Wire `ignore_spaces`:
- Already have `normalize_nospace()` fallback — make it try nospace FIRST when `ignore_spaces=True`

Wire `ignore_punctuation`:
- Add `normalize_no_punctuation()` (graft from QT text_utils.py) as additional fallback
