Create `server/tools/ldm/routes/codex_items.py` with an APIRouter at prefix `/codex/items`, tags=["Codex Items"]:

2. **GET `/` (list items)** — response_model=ItemListResponse:
   - Query params: `offset: int = Query(0, ge=0)`, `limit: int = Query(50, ge=1, le=200)`, `group: Optional[str] = Query(None)`, `q: Optional[str] = Query(None)`, `lang: str = Query("eng")`
   - Call `mega = get_mega_index()`; `mega.all_entities("item")` to get Dict[str, ItemEntry]
   - If `group` param set: collect all descendant group strkeys by walking `get_item_group_tree()` recursively from the given group, then filter items whose `group_key` is in that set
   - If `q` param set: filter items where q.lower() appears in entry.name.lower(), entry.desc.lower(), entry.strkey.lower(), or translated name (call get_translation for first "_NAME" stringid)
   - For each surviving item, build ItemCardResponse:
     - `name_kr` = entry.name
     - `name_translated`: get stringids via `mega.entity_stringids("item", strkey)`, find first ending with `_NAME`, call `mega.get_translation(sid, lang)`
     - `image_url`: if `mega.get_knowledge(entry.knowledge_key)` has `.ui_texture_name`, set `f"/api/ldm/mapdata/thumbnail/{ui_texture_name}"`, else None
     - `group_name`: lookup group_tree[entry.group_key].group_name, default None
   - Sort by name_kr, paginate with offset/limit, compute has_more
   - Wrap in try/except, log errors with logger, return 500 on failure

3. **GET `/groups` (group tree)** — response_model=ItemGroupTreeResponse:
   - Call `mega.get_item_group_tree()` to get Dict[str, ItemGroupNode]
   - Build nested ItemGroupTreeNode list: roots = nodes where parent_strkey == ""
   - Recursive helper to build children from child_strkeys
   - Count total_groups and total_items (sum of all item_strkeys across all nodes)

4. **GET `/{strkey}` (item detail)** — response_model=ItemDetailResponse:
   - Call `mega.get_item(strkey)` — raise HTTPException(404) if None
   - Knowledge Pass 0: if entry.knowledge_key, get knowledge entry, then find all knowledge entries that reference this key via `mega.find_by_knowledge_key(entry.knowledge_key)` filtered to entity_type=="knowledge"
   - Knowledge Pass 1: Direct knowledge from `mega.get_knowledge(entry.knowledge_key)` as single entry
   - Knowledge Pass 2: `mega.find_by_korean_name(entry.name)` filtered to entity_type=="knowledge", excluding strkeys already in pass 0/1
   - inspect_data: map entry.inspect_entries tuples to InspectDataEntry objects
   - related_entities: `mega.find_by_knowledge_key(entry.knowledge_key)` — collect strkeys excluding self
   - image_url and name_translated same logic as list endpoint
