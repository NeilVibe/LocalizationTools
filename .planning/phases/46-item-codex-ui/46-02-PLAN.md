**ItemCodexPage.svelte** — Following CodexPage.svelte patterns exactly:

1. **State** (all Svelte 5 Runes):
   - `let groups = $state([])` — root ItemGroupTreeNode array from /groups endpoint
   - `let activeGroup = $state(null)` — currently selected group strkey (null = All Items)
   - `let items = $state([])` — current page of ItemCardResponse objects
   - `let selectedItem = $state(null)` — selected item for detail view (full ItemDetailResponse)
   - `let searchQuery = $state("")` — debounced search text
   - `let loadingGroups = $state(true)`, `let loadingList = $state(false)`, `let loadingDetail = $state(false)`
   - `let apiError = $state(null)`, `let failedImages = $state(new Set())`
   - `let currentPage = $state(0)`, `let hasMore = $state(true)`, `let loadingMore = $state(false)`
   - `const PAGE_SIZE = 50`

2. **Data fetching**:
   - `fetchGroups()`: GET /api/ldm/codex/items/groups, populate `groups` with top-level groups. Add an "All" pseudo-tab at position 0 (activeGroup=null).
   - `fetchItems(page, group, query)`: GET /api/ldm/codex/items with offset/limit/group/q params. Build URL with URLSearchParams. On page 0, replace items; on page > 0, append. Use AbortController for stale request cancellation.
   - `fetchItemDetail(strkey)`: GET /api/ldm/codex/items/{strkey}, set selectedItem.
   - `loadMore()`: if hasMore and not loadingMore, fetch next page.
   - All fetches use `getAuthHeaders()` from `$lib/utils/api.js`.

3. **Search** (per ITEM-04):
   - Debounced search input (300ms) using `$effect` + `setTimeout`
   - When searchQuery changes, reset to page 0 and call fetchItems with q param
   - Search bar at top, same styling as CodexPage `.codex-search`

4. **Group tabs** (per ITEM-02):
   - Horizontal tab bar below search, same pattern as CodexPage `.codex-tabs`
   - "All" tab (no group filter) + one tab per root group showing `group_name (item_count)`
   - On tab click: set activeGroup, reset page, fetchItems with group param
   - Active tab gets `.active` class with blue border-bottom

5. **Card grid** (per ITEM-01):
   - Reuse existing CodexCard component. Transform each ItemCardResponse to CodexCard's expected entity shape:
     ```javascript
     { strkey: item.strkey, name: item.name_kr, entity_type: 'item',
       description: item.name_translated
         ? item.name_translated + (item.desc_kr ? '\n' + item.desc_kr : '')
         : item.desc_kr,
       image_texture: null,
       ai_image_url: item.image_url || null,
       related_entities: [] }
     ```
     Note: set `ai_image_url` to the `image_url` from the API (which is the DDS thumbnail path). If image_url is null, CodexCard falls back to PlaceholderImage.
   - Keep entity_type='item' so existing cyan badge shows
   - Grid layout: `grid-template-columns: repeat(auto-fill, minmax(220px, 1fr))` same as CodexPage
   - InfiniteScroll sentinel at bottom with `onloadmore={loadMore}`

6. **Detail panel** (per ITEM-03):
   - On card click: call fetchItemDetail(strkey), show ItemCodexDetail component
   - Back button at top (same pattern as CodexPage)
   - Pass full ItemDetailResponse to ItemCodexDetail

7. **Page header**: Use PageHeader component with `icon={Catalog}` (from carbon-icons-svelte) and title "Item Codex"

8. **Styles**: Copy CodexPage's CSS structure (`.codex-page`, `.codex-search`, `.codex-tabs`, `.entity-grid`, etc.) but rename classes to `.item-codex-*` to avoid conflicts. Same glassmorphism, same dark theme variables.

9. **onMount**: Call fetchGroups(), then fetchItems(0, null, "").

**ItemCodexDetail.svelte** — Item-specific detail panel:

1. **Props**: `let { item = null, onback = () => {}, onsimilar = () => {} } = $props()`

2. **Layout** (following CodexEntityDetail pattern):
   - Top: Back button (ArrowLeft icon)
   - Left column: DDS image (use `<img src="{apiBase}{item.image_url}">` if available, else PlaceholderImage)
   - Right column: Korean name (h2), translated name (subtitle), description, group badge, source file

3. **Knowledge resolution tabs** (ITEM-03 core feature):
   - Tab bar with 4 tabs: "Knowledge" (Pass 0+1), "Related" (Pass 2), "InspectData", "Info"
   - "Knowledge" tab: Show pass_0 entries first (inline children), then pass_1 entries (direct KnowledgeKey). Each entry renders as a card with name bold, desc below, source in small gray text.
   - "Related" tab: pass_2 entries (name-matched knowledge). Same card layout.
   - "InspectData" tab: inspect_data entries. Each shows desc, knowledge_name, knowledge_desc, source.
   - "Info" tab: knowledge_key, group_key, group_name, source_file, strkey.
   - If a tab has 0 entries, show "(0)" in the tab label and display "No entries" in content area.
   - Use Svelte 5 `$state` for activeTab, default to "Knowledge".

4. **Related entities**: Show related_entities as clickable badges (same pattern as CodexEntityDetail's related-badge). On click, call onsimilar(strkey).

5. **Styles**: Follow CodexEntityDetail CSS patterns. Glassmorphism card container, dark theme.

**CRITICAL Svelte 5 rules**:
- Use `$state`, `$derived`, `$effect` ONLY (no Svelte 4 `export let`, no `$:`)
- Use `$props()` for component props
- Always use keys in `{#each}` loops: `{#each items as item (item.strkey)}`
- Use `onclick` not `on:click`
