    These files have NO createEventDispatcher (already handled by Plans 01/02) but DO have remaining `on:` directives and `e.detail` access that needs conversion.

    **Carbon component on: directives that ARE EXEMPT (keep as-is):**
    - `<Button on:click={handler}>` -- Carbon Button requires on:click
    - `<Modal on:close={handler}>` / `<ComposedModal on:close>` -- Carbon Modal
    - `on:click:button--primary` / `on:click:button--secondary` -- Carbon ComposedModal
    - `<InlineNotification on:close>` -- Carbon Notification
    - `<Form on:submit>` -- Carbon Form
    - `<Dropdown on:select>` -- Carbon Dropdown
    - `<Search on:input>` / `<Search on:clear>` / `<Search on:focus>` / `<Search on:blur>` -- Carbon Search
    - `<Slider on:change>` -- Carbon Slider
    - `<Toggle on:toggle>` -- Carbon Toggle
    - `<Checkbox on:check>` -- Carbon Checkbox
    - `<RadioButtonGroup on:change>` -- Carbon RadioButtonGroup
    - `<ContentSwitcher on:change>` -- Carbon ContentSwitcher
    - `<Pagination on:update>` -- Carbon Pagination
    - `<FileUploaderButton on:change>` -- Carbon FileUploader
    - `<TextInput on:keydown>` / `<TextInput on:keypress>` -- Carbon TextInput

    **Directives to CONVERT (custom Svelte components and DOM elements):**
    - `<form on:submit|preventDefault={handler}>` -- native HTML form -> `<form onsubmit={handler}>` (add preventDefault in handler if needed, or use `onsubmit={(e) => { e.preventDefault(); handler(e); }}`)
    - `<svelte:window on:keydown={handler}>` -- -> `<svelte:window onkeydown={handler}>`
    - `<input on:keydown>` / `<input on:keypress>` -- native HTML -> `onkeydown` / `onkeypress`

    **e.detail conversions:**
    - `GridColumnsModal`: `on:check={(e) => { showIndex = e.detail; ... }}` -- This is Carbon Checkbox's on:check, which uses e.detail. Keep on:check (Carbon EXEMPT) but e.detail stays as Carbon provides it.
    - `PreferencesModal`: `on:toggle={(e) => handleFontWeightChange(e.detail.toggled)}` -- Carbon Toggle, e.detail stays.
    - `ReferenceSettingsModal`: Check if e.detail is from Carbon or custom.
    - `QuickSearch`: `on:toggle={(e) => toggleReference(e.detail.toggled)}` -- Carbon Toggle, e.detail stays.

    **Key insight: If the on: directive is on a Carbon component, BOTH the on: AND the e.detail stay. Only convert on: and e.detail for custom Svelte components and native DOM elements.**

    For each file:
    1. Read the file first
    2. Identify which on: directives are on Carbon components (KEEP) vs DOM elements/custom components (CONVERT)
    3. Convert only the non-Carbon ones
    4. Convert any `<svelte:window on:event>` to `<svelte:window onevent>`
    5. Convert native DOM `on:submit`, `on:keydown`, `on:keypress` to `onsubmit`, `onkeydown`, `onkeypress`
