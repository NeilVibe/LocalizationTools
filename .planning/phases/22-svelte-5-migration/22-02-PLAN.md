  <action>
    For each file, apply the Svelte 5 callback migration pattern:

    1. Remove `import { createEventDispatcher } from "svelte"` and `const dispatch = createEventDispatcher()`
    2. Add callback props to `$props()` destructuring: `let { ..., onEventName = undefined } = $props()`
    3. Replace every `dispatch('eventName', data)` with `onEventName?.(data)`

    **TMDataGrid:** Dispatches applyTM, plus has `<svelte:window on:keydown>` -- convert to `onkeydown`. Also has on:click on Button (Carbon EXEMPT).
    **TMViewer:** Dispatches updated, close. Has `<svelte:window on:keydown>` -- convert to `onkeydown`. Has on:close on Modal (Carbon EXEMPT).
    **TMTab:** Dispatches applyTM, tmSelect.
    **TMQAPanel:** Dispatches dismissQA, runQA.
    **TMUploadModal:** Dispatches uploaded, close.
    **FilePickerDialog:** Dispatches select, close.
    **PretranslateModal:** Dispatches close.
    **AccessControl:** Dispatches login, logout.

    Read each file first to identify ALL dispatch calls. Convert `<svelte:window on:keydown={handler}>` to `<svelte:window onkeydown={handler}>`.

    Naming convention: `on` + PascalCase event name. Example: `dispatch('applyTM', data)` -> `onApplyTM?.(data)`.

    IMPORTANT: Keep Carbon component `on:` directives (on:click on Button, on:close on Modal, on:change on Dropdown, etc.) -- these are EXEMPT per REQUIREMENTS.md.
  </action>
