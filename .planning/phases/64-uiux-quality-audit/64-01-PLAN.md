**1. Preview retry button (per UIUX-03):**
In the preview phase, when `previewError` is truthy, add a "Retry Preview" Button (kind="tertiary") next to the error notification that calls `runPreview()` directly. This avoids the user having to go Back then click Preview again.

```svelte
{:else if previewError}
  <InlineNotification kind="error" title="Preview failed" subtitle={previewError} hideCloseButton lowContrast />
  <div class="modal-actions">
    <Button kind="secondary" onclick={goBackToConfigure}>Back</Button>
    <Button kind="tertiary" onclick={runPreview}>Retry Preview</Button>
  </div>
```
Remove the duplicate Back/Execute buttons section that shows below (the existing modal-actions at the bottom of preview phase should only render when `previewResult` exists and no error).

**2. Zero-match guard (per UIUX-03):**
Add a derived:
```svelte
let hasZeroMatches = $derived(previewResult && (previewResult.total_matched ?? 0) === 0);
```
When `hasZeroMatches` is true, show an InlineNotification (kind="info") saying "No matching entries found. Check your match type and source files." and disable the Execute Merge button. Update the Execute button disabled condition:
```svelte
disabled={previewLoading || hasPreviewErrors || !previewResult || hasZeroMatches}
```

**3. AbortController for execute cancel (per UIUX-03):**
Add state:
```svelte
let abortController = $state(null);
```
In `executeMerge()`, create an AbortController before the fetch:
```js
abortController = new AbortController();
const response = await fetch(url, { ...opts, signal: abortController.signal });
```
Add a `cancelMerge()` function:
```js
function cancelMerge() {
  if (abortController) {
    abortController.abort();
    abortController = null;
  }
  progressMessages = [...progressMessages, '[CANCELLED] Merge cancelled by user'];
  executing = false;
}
```
In the catch block of `executeMerge()`, check for AbortError:
```js
catch (err) {
  if (err.name === 'AbortError') {
    progressMessages = [...progressMessages, '[CANCELLED] Merge cancelled by user'];
  } else {
    progressMessages = [...progressMessages, `[Error] Connection lost: ${err.message}`];
  }
}
```
In the execute phase template, add a Cancel button when `executing` is true:
```svelte
{#if executing}
  <div class="modal-actions">
    <Button kind="danger--tertiary" onclick={cancelMerge}>Cancel Merge</Button>
  </div>
{/if}
```
Keep the existing "Back to Configure" button for when `!executing && phase === 'execute'`.

Clean up `abortController` in the reset-on-open effect by adding `abortController = null;`.

**4. Do NOT change:** The passiveModal behavior, the SSE parsing logic, the configure phase, or the styling (those are fine).

**1. Execute error recovery (per UIUX-03):**
When the execute phase ends with errors (SSE 'error' event received, or response not OK), the user currently sees errors in the log but has no clear recovery path. Add:

After the progress log in the execute phase, when `!executing && phase === 'execute'` (meaning execution ended but didn't reach 'done'), show an InlineNotification:
```svelte
{#if !executing && phase === 'execute'}
  <InlineNotification
    kind="error"
    title="Merge did not complete"
    subtitle="The merge was interrupted. You can retry or go back to reconfigure."
    hideCloseButton
    lowContrast
  />
  <div class="modal-actions">
    <Button kind="secondary" onclick={goBackToConfigure}>Back to Configure</Button>
    <Button kind="danger" onclick={executeMerge}>Retry Merge</Button>
  </div>
{/if}
```
This replaces the existing `{#if !executing && phase === 'execute'}` block which only has "Back to Configure".

**2. Adaptive done-phase messaging (per UIUX-03):**
Add a derived to detect if errors occurred during execution:
```svelte
let hadExecutionErrors = $derived(progressMessages.some(m => m.startsWith('[ERROR]') || m.startsWith('[Error]')));
```
In the done phase, change the InlineNotification to be conditional:
```svelte
{#if hadExecutionErrors}
  <InlineNotification
    kind="warning"
    title="Merge Completed with Issues"
    subtitle="Some files encountered errors during processing. Review the execution log below."
    hideCloseButton
    lowContrast
  />
{:else}
  <InlineNotification
    kind="success"
    title="Merge Complete"
    subtitle="All files have been processed successfully."
    hideCloseButton
    lowContrast
  />
{/if}
```
When `hadExecutionErrors` is true, auto-expand the log details by using `open` attribute on the `<details>` element:
```svelte
<details class="log-details" open={hadExecutionErrors}>
```
