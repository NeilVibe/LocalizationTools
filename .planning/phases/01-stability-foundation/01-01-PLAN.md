Create `tests/stability/test_schema_drift.py` with these test functions:

1. **`parse_create_tables(sql_content: str) -> dict`** helper:
   - Use `sqlparse` to parse SQL content
   - Extract table name and column definitions from each CREATE TABLE statement
   - Return `{table_name: {col_name: col_type_string, ...}, ...}`
   - Handle both `CREATE TABLE` and `CREATE TABLE IF NOT EXISTS`

2. **`test_offline_schema_covers_all_ldm_tables()`**:
   - Import TABLE_MAP from `server.repositories.sqlite.base`
   - Parse `server/database/offline_schema.sql`
   - For each entry in TABLE_MAP, assert the offline_name table exists in the parsed schema
   - Report which tables are missing if any

3. **`test_offline_only_columns_complete()`**:
   - Import OFFLINE_ONLY_COLUMNS from `server.repositories.sqlite.base`
   - Parse both `server/database/offline_schema.sql` and `server/database/database_schema.sql`
   - For each table pair (offline vs PostgreSQL equivalent), find columns that exist in offline but NOT in PostgreSQL
   - Assert those extra columns are ALL present in OFFLINE_ONLY_COLUMNS (catches missing allowlist entries)
   - Assert ALL entries in OFFLINE_ONLY_COLUMNS actually ARE extra columns (catches stale allowlist entries)

4. **`test_no_postgresql_specific_sql_in_repos()`**:
   - Grep all `.py` files under `server/repositories/` for PostgreSQL-specific constructs:
     - `RETURNING` clause
     - `ILIKE` (case-insensitive LIKE, PostgreSQL-only)
     - `::` type casts (e.g., `::text`, `::integer`)
     - `NOW()` function (SQLite uses `datetime('now')`)
     - `ARRAY[` or `array_agg` (PostgreSQL array operations)
   - Assert zero matches in any repository file
   - Exclude `__pycache__` directories
   - Use Python `pathlib.Path.rglob("*.py")` + regex, NOT subprocess

5. **`test_sqlite_pragmas(db_mode, clean_db)`**:
   - Skip if db_mode is ONLINE
   - Connect to the clean_db SQLite file
   - Assert `PRAGMA foreign_keys` returns 1
   - Assert `PRAGMA journal_mode` returns "wal" (if the conftest sets WAL mode; otherwise document that WAL is not set and note for future)
   - Close connection

6. **`test_server_local_schema_matches_offline_structure()`**:
   - Create a temp SQLite DB, initialize via SQLAlchemy `Base.metadata.create_all`
   - Parse the resulting schema (via `sqlite_master` query)
   - Compare against `offline_schema.sql` parsed tables
   - Column names should match (ignoring OFFLINE_ONLY_COLUMNS)
   - This catches Pitfall 5 from research (two different init paths producing different schemas)

Use `from __future__ import annotations` at top. Mark all tests with `@pytest.mark.stability`.
