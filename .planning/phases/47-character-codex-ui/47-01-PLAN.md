  <action>
Create `server/tools/ldm/routes/codex_characters.py` with router prefix `/codex/characters`.

**Helper: _parse_use_macro(use_macro: str) -> Tuple[Optional[str], Optional[str]]**
Split on "_" and extract race + gender from known patterns.
Pattern: "Macro_NPC_Human_Male" -> race="Human", gender="Male"
Logic: split by "_", find known race tokens (Human, Elf, Dwarf, Giant, Shai, etc.), find gender tokens (Male, Female). Return (race, gender). If unparseable, return (None, None).

**Helper: _extract_category(source_file: str) -> str**
Strip "characterinfo_" prefix and ".staticinfo.xml" suffix from source_file.
Example: "characterinfo_npc.staticinfo.xml" -> "NPC" (uppercased)
Example: "characterinfo_monster.staticinfo.xml" -> "MONSTER"
If no match, use source_file stem uppercased.

**Helper: _build_character_card(strkey, mega, lang) -> CharacterCardResponse**
Same pattern as _build_item_card in codex_items.py:
- Get CharacterEntry from mega.get_character(strkey)
- Translated name: mega.entity_stringids("character", strkey), find _NAME suffix, mega.get_translation()
- Image URL: if entry.knowledge_key -> mega.get_knowledge() -> knowledge.ui_texture_name -> `/api/ldm/mapdata/thumbnail/{name}`
- Fallback image: if no knowledge image and entry.ui_icon_path, use that
- Parse race/gender from entry.use_macro via _parse_use_macro
- Parse category from entry.source_file via _extract_category

**Endpoint 1: GET /codex/characters/categories** (response_model=CharacterCategoryResponse)
- mega.all_entities("character"), group by _extract_category(entry.source_file)
- Return sorted category list with counts + total_characters
- MUST be defined BEFORE /{strkey} to avoid route conflict

**Endpoint 2: GET /codex/characters/{strkey}** (response_model=CharacterDetailResponse)
- Same 3-pass knowledge resolution as codex_items.py get_item_detail
- Pass 0: entities referencing same knowledge_key (filter to "knowledge" type)
- Pass 1: Direct knowledge_key resolution
- Pass 2: Korean name matches (excluding seen)
- Include age, job, use_macro, race, gender fields
- 404 if character not found

**Endpoint 3: GET /codex/characters/** (response_model=CharacterListResponse)
- Query params: offset (default 0), limit (default 50, max 200), category (Optional), q (Optional), lang (default "eng")
- Category filtering: filter entries where _extract_category(entry.source_file) == category (case-insensitive)
- Text search (q): search across entry.name, entry.desc, entry.strkey, entry.use_macro, entry.age, entry.job, and translated name (per CHAR-04)
- Sort by Korean name, paginate, return CharacterListResponse
