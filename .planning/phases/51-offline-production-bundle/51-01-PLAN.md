    1. In server/database/offline.py, in the _init_schema_sync method, add AFTER the schema creation SQL:
       ```python
       conn.execute("PRAGMA journal_mode=WAL;")
       conn.execute("PRAGMA busy_timeout=10000;")
       ```
       This prevents "database is locked" errors during concurrent reads+writes in offline-only mode (Pitfall 3 from PITFALLS.md). WAL allows concurrent readers while one writer is active. busy_timeout=10000ms (10s) prevents immediate failures under load.

    2. In server/database/server_sqlite.py:
       - In _get_connection(), add after conn = sqlite3.connect(self.db_path):
         ```python
         conn.execute("PRAGMA journal_mode=WAL;")
         conn.execute("PRAGMA busy_timeout=10000;")
         ```
       - In _get_async_connection(), add the same PRAGMAs after opening connection:
         ```python
         await conn.execute("PRAGMA journal_mode=WAL;")
         await conn.execute("PRAGMA busy_timeout=10000;")
         ```
