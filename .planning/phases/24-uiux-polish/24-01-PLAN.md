- Replace the outer `<svg>` with a `<div class="placeholder-image">` container
- Use CSS for the rounded rect background (background-color + border-radius)
- Render the Carbon `<IconComponent>` directly in a centered flex container (no foreignObject)
- Render the entity name as a `<span>` below the icon (not SVG `<text>`)
- Keep the same visual dimensions (~160x120 aspect ratio via CSS aspect-ratio or padding)
- Preserve all existing props ($props: entityType, entityName) and derived state (IconComponent, displayName)
- Use CSS variables: `var(--cds-layer-02)` for background, `var(--cds-text-03)` for text/icon color
- Keep `.placeholder-image { width: 100%; max-width: 200px; border-radius: 4px; }`

**CodexPage.svelte (UX-03):** Update the entity card image section to show PlaceholderImage on 404 instead of just hiding the broken image.

1. Add import: `import PlaceholderImage from "$lib/components/ldm/PlaceholderImage.svelte";`
2. Track failed images with a reactive Set: `let failedImages = $state(new Set());`
3. Replace the card-image content (lines ~254-264). For each entity card:
   ```svelte
   <div class="card-image">
     {#if entity.image_texture && !failedImages.has(entity.strkey)}
       <img
         src="{API_BASE}/api/ldm/mapdata/thumbnail/{entity.image_texture}"
         alt={entity.name}
         class="card-thumb"
         onerror={() => {
           const next = new Set(failedImages);
           next.add(entity.strkey);
           failedImages = next;
         }}
       />
     {:else}
       <PlaceholderImage entityType={entity.entity_type} entityName={entity.name} />
     {/if}
   </div>
   ```
4. This ensures entities with no image_texture AND entities whose images return 404 both show the PlaceholderImage component.
