Rewrite the `handleFileSelect` function in GameDevPage.svelte:

1. **Remove the entire upload-path fetch block** (lines 56-83 approximately). The `/api/ldm/files/upload-path` endpoint does not exist. The FileExplorerTree's browse response already provides all needed file metadata.

2. **Remove the FormData creation** and the `fileData` variable. Instead, build the gridFile object directly from the `file` parameter (which comes from FileExplorerTree's onFileSelect callback and contains `{name, path, entity_count}`).

3. **Replace the gridFile construction** to use `file.path` as the ID instead of `fileData.id || Date.now()`. The file path is a unique, stable identifier. Use a deterministic ID like a hash or just the path string itself.

New handleFileSelect should be approximately:
```javascript
async function handleFileSelect(file) {
  selectedGameDevFile = file;
  fileLoading = true;
  dynamicColumns = null;

  try {
    // Fetch dynamic columns for the selected XML file
    try {
      const colResponse = await fetch(`${API_BASE}/api/ldm/gamedata/columns`, {
        method: 'POST',
        headers: { ...getAuthHeaders(), 'Content-Type': 'application/json' },
        body: JSON.stringify({ xml_path: file.path })
      });

      if (colResponse.ok) {
        const colData = await colResponse.json();
        if (colData.columns && colData.columns.length > 0) {
          const cols = {
            row_num: { key: "row_num", label: "#", width: 60, prefKey: "showIndex" },
            node_name: { key: "source", label: "Node", width: 200, always: true }
          };
          for (const hint of colData.columns) {
            cols[hint.key] = {
              key: hint.key,
              label: hint.label,
              width: hint.editable ? 250 : 150,
              always: true
            };
          }
          cols.children_count = { key: "children_count", label: "Children", width: 100, always: true };
          dynamicColumns = cols;
          logger.success('Dynamic columns loaded', { count: colData.columns.length });
        }
      } else {
        logger.warning('Dynamic columns fetch failed, using defaults');
      }
    } catch (err) {
      logger.warning('Dynamic columns unavailable', { error: err.message });
    }

    // Set file in openFile store -- use path as stable ID (no Date.now() fallback)
    const gridFile = {
      id: file.path,
      name: file.name,
      path: file.path,
      format: 'xml',
      file_type: 'gamedev',
      entity_count: file.entity_count
    };
    openFile.set(gridFile);

    logger.success('Game Dev file selected', { name: file.name, entity_count: file.entity_count });
  } catch (err) {
    logger.error('Failed to load game dev file', { error: err.message });
  } finally {
    fileLoading = false;
  }
}
```

  <name>Task 2: Fix tree refresh to use reload instead of remount flicker (FIX-08)</name>
  <files>locaNext/src/lib/components/pages/GameDevPage.svelte</files>
  <action>
Fix the `refreshTree` function in GameDevPage.svelte. Currently it uses a hack:
```javascript
function refreshTree() {
  const current = activePath;
  activePath = '';
  setTimeout(() => { activePath = current; }, 50);
}
```

This causes a visible flicker because it unmounts and remounts the entire FileExplorerTree component.

Replace with a bind:this approach that calls a reload method on the FileExplorerTree component:

1. Add a `let treeRef = $state(null);` state variable.
2. Bind to the FileExplorerTree: `bind:this={treeRef}`.
3. Update refreshTree to call the reload method:
```javascript
function refreshTree() {
  if (treeRef?.reload) {
    treeRef.reload();
  } else {
    // Fallback: re-apply path (still better than empty-then-restore)
    const current = activePath;
    activePath = current + ' ';
    tick().then(() => { activePath = current; });
  }
  logger.userAction('Game Dev tree refreshed');
}
```

4. Check if FileExplorerTree already exports a `reload` method. If not, the fallback approach using Svelte's `tick()` is acceptable -- it's still better than the setTimeout flicker because tick is synchronous-ish and avoids the 50ms empty state.

Import `tick` from 'svelte' if needed.
