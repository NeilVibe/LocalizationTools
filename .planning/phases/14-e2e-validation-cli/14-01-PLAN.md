Add 4 new command functions to scripts/locanext_cli.py (after the existing E2E section, before the CLI Parser section):

1. **cmd_merge(target_id, source_id, mode="cascade", threshold=0.85, is_cjk=False):**
   - POST /api/ldm/files/{target_id}/merge with JSON body {source_file_id, match_mode, threshold, is_cjk}
   - Print MergeResponse fields: matched, skipped, total, match_type_counts, rows_updated
   - Use existing `api()` helper

2. **cmd_gamedev_merge(file_id, max_depth=3, output_path=None):**
   - POST /api/ldm/files/{file_id}/gamedev-merge with JSON body {max_depth}
   - Print change counts: total_nodes, changed_nodes, added_nodes, removed_nodes, modified_attributes
   - If output_path provided, decode base64 output_xml and write to file
   - If no output_path, default to f"gamedev_merge_{file_id}.xml"

3. **cmd_export(file_id, format="xml", output_path=None, status_filter=None):**
   - GET /api/ldm/files/{file_id}/download with query params {status_filter} if provided
   - Use requests.get directly (not api() helper) since response is binary/streaming, not JSON
   - Save response content to output_path (default: f"export_{file_id}.{format}")
   - Print file size and path

4. **cmd_detect(file_id):**
   - GET /api/ldm/files/{file_id} via api() helper
   - Extract file_type from response (defaults to "translator")
   - Print: "File {id}: {name} -> {file_type}"

Update the main() CLI parser to route these commands:

- `export <file_id> [--format xml|xlsx|txt] [--output path] [--status reviewed|translated|all]`
- `detect <file_id>`

For the CLI parser: continue using sys.argv parsing (matching the existing pattern in main()), NOT argparse. Keep it consistent with the existing code style.
