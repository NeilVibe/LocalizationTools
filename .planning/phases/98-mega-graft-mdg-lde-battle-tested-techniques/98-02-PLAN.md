**Fix 1: Resize delta (GRAFT-05)**

In `GameDataContextPanel.svelte`, find the `handleResize` function (around line 385-388):
```javascript
// CURRENT (WRONG): const delta = resizeStartX - e.clientX;
// Panel is on the RIGHT side. Dragging LEFT (decreasing clientX) should INCREASE width.
// So delta = resizeStartX - e.clientX is CORRECT for a right-side panel.
```

WAIT - re-examine: The panel is on the RIGHT side of the screen. The resize handle is on the LEFT edge of the panel. Dragging LEFT (e.clientX decreases) means the user wants the panel WIDER. So `delta = resizeStartX - e.clientX` gives a POSITIVE delta when dragging left, and `resizeStartWidth + delta` makes it wider. This is CORRECT for right-side panels.

Actually, the research says it is inverted. Let me re-read: "resizeStartX - e.clientX should be e.clientX - resizeStartX". Test: if panel is on RIGHT and drag handle is on LEFT edge, dragging LEFT means clientX < resizeStartX. We want width to INCREASE. With `resizeStartX - e.clientX`, delta is positive, width increases. That sounds correct.

BUT: the research explicitly confirms the behavior is wrong in practice. The issue may be that the panel uses `width` but is positioned with `flex` or absolute positioning where increasing width should correspond to the opposite delta direction.

**Resolution:** Change line 387 from:
```javascript
const delta = resizeStartX - e.clientX;
```
to:
```javascript
const delta = e.clientX - resizeStartX;
```
Per GRAFT-05 research which confirmed drag direction is inverted in practice. If this turns out wrong after testing, it can be reverted with a sign flip.

```svelte
{#if entity?.audio?.wem_path}
  ...
```
