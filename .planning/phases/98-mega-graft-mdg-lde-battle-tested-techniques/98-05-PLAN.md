Add to `gamedata.py`:
```python
from ..services.mega_index import get_mega_index

@router.post("/trigger-mega-build")
async def trigger_mega_build(
    _user=Depends(get_current_active_user_async),
):
    """Trigger MegaIndex build after gamedata folder is loaded. Non-blocking response."""
    try:
        mi = get_mega_index()
        if mi.is_built:
            return {"status": "already_built", "stats": mi.stats()}
        result = mi.build()
        return {"status": "success", "stats": mi.stats()}
    except Exception as e:
        logger.error(f"MegaIndex auto-build failed: {e}")
        return {"status": "error", "error": str(e)}
```

Alternatively, if the existing `/mega/build` endpoint is sufficient, the frontend can just call that directly. Check if it requires auth. If it does not have auth, add the `_user=Depends(get_current_active_user_async)` guard.

The key point: DO NOT call mega build inside the folder_tree endpoint itself (that would block folder loading). The frontend should call it as a separate fire-and-forget after the folder tree loads.
