    **RegionCodexMap.svelte** — Interactive d3-zoom map rendering WorldPosition coordinates:
    - Props: `regions` (array of {strkey, name, node_type, world_position, has_position}), `selectedStrkey` (string|null), `onNodeClick` (callback), `onNodeHover` (callback), `onNodeLeave` (callback)
    - Use Svelte 5 `$props()` for all props
    - SVG with d3-zoom (copy pattern from MapCanvas.svelte):
      - `import { zoom, zoomIdentity } from "d3-zoom"` and `import { select } from "d3-selection"`
      - SVG_SIZE = 1000, PADDING = 50
      - `transform = $state({ x: 0, y: 0, k: 1 })`
      - Bind svgElement, init zoom in onMount, cleanup in onDestroy
    - Coordinate mapping: WorldPosition uses (x, z) for 2D top-down view (y = elevation, ignore)
      - Compute bounds from all regions with world_position: minX, maxX, minZ, maxZ
      - Map function: `mapX(wx) = PADDING + ((wx - minX) / (maxX - minX)) * (SVG_SIZE - 2*PADDING)`
      - Map function: `mapZ(wz) = PADDING + ((wz - minZ) / (maxZ - minZ)) * (SVG_SIZE - 2*PADDING)`
      - Handle edge case: if all same position, center single node
    - Node rendering: SVG circles with size 8 (normal) / 12 (selected), color by node_type using NODE_COLORS from MapCanvas pattern (Main=#0f62fe, Sub=#4589ff, Dungeon=#da1e28, Fortress=#8a3ffc, default=#6f6f6f)
    - Selected node: stroke highlight (white, 3px), slightly larger radius
    - Labels: SVG text below each node, show region name, font-size 10px, fill var(--cds-text-01)
    - Parchment background: CSS radial-gradient pattern (copy from MapCanvas.svelte style)
    - Mouse events: on circle click -> onNodeClick(region), hover -> onNodeHover, leave -> onNodeLeave
    - Minimap in bottom-right corner (optional, copy from MapCanvas if straightforward)

    **RegionCodexDetail.svelte** — Detail panel for selected region:

    - Layout (follow AudioCodexDetail / ItemCodexDetail pattern):

      - node_type badge (colored Tag component or span with background)
      - WorldPosition display: "Position: (x, y, z)" formatted to 1 decimal
      - Faction breadcrumb: "FactionGroup > Faction"
      - Image (if image_url) using img tag with /api/ldm prefix
      - Description (desc_kr) if present

        - Tab 1: "Direct" (pass_1 entries)
        - Tab 2: "Related" (pass_2 entries)
