<objective>
Fix the grid's default background color so cells without a special status appear neutral/white-ish rather than appearing amber/yellow.

Purpose: During v9.0 Windows testing, the grid looked visually dominated by amber/yellow. The amber status-translated cells (`rgba(198,163,0,0.12)`) bleed visually when many rows are translated. The fix makes the default row background more distinctly neutral — a light-grey-on-dark-theme tone — so uncolored cells are clearly neutral and the amber/teal status accents read as intentional highlights rather than the default state.

Output: One CSS change in VirtualGrid.svelte. Default row/cell background becomes a subtle neutral (slightly lighter than `#1a1a1a`). Status colors unchanged.
</objective>

  <action>
The goal is to make the default (no-status) grid row background clearly neutral, so it does not visually compete with or blend into the amber status-translated color.

**Current state (do NOT change these values):**
- `.virtual-row { background: var(--cds-layer-01); }` — resolves to `#1a1a1a` (very dark grey)
- `.cell.target.status-translated { background: rgba(198, 163, 0, 0.12); }` — amber tint
- `.cell.target.status-reviewed { background: rgba(0, 157, 154, 0.15); }` — teal tint
- `.cell.target.status-approved { background: rgba(0, 157, 154, 0.15); }` — teal tint

**The fix — two CSS changes:**

1. On `.virtual-row`, add a CSS custom property `--row-bg-default` and change the background to a subtly lighter neutral tone. The dark theme uses `#1a1a1a` as its base — use `#222222` (a step lighter, clearly readable as neutral, still dark-theme appropriate). This gives a clear visual "baseline" that is not amber. Change:

```css
.virtual-row {
    position: absolute;
    left: 0;
    right: 0;
    display: flex;
    border-bottom: 1px solid rgba(255, 255, 255, 0.06);
    background: #222222; /* UI-103: neutral default — not yellow */
    transition: background-color 0.15s ease;
    box-sizing: border-box;
}
```

2. To make the amber status cells stand out MORE against the neutral row base (so they clearly read as "needs attention"), slightly increase the amber opacity from 0.12 to 0.16. This reinforces the semantic signal. Change ONLY the `status-translated` background value:

```css
.cell.target.status-translated {
    background: rgba(198, 163, 0, 0.16); /* amber - needs confirmation (UI-103: increased from 0.12 for visibility) */
    border-left: 3px solid #c6a300;
}
```

Do NOT change status-reviewed or status-approved. Do NOT change .virtual-row hover/selected states. Do NOT change .cell.source background. Only these two targeted CSS property values change.

Add comment `/* UI-103: neutral default — not yellow */` inline on the .virtual-row background line for traceability.
  </action>
