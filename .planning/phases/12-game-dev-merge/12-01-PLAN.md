<feature>
  <name>GameDevMergeService: Parallel Tree Walk Diff + In-Place Apply</name>
  <files>server/tools/ldm/services/gamedev_merge.py, tests/unit/ldm/test_gamedev_merge.py, tests/fixtures/xml/gamedev_modified.xml</files>
  <behavior>
    GMERGE-01 (diff detection):
    - Given original XML and rows where one attribute changed, diff returns exactly 1 NodeChange with change_type=MODIFIED
    - Given original XML and rows where all attributes match, diff returns 0 changes (all UNCHANGED)
    - Given original XML and rows with a new node added, diff returns 1 NodeChange with change_type=ADDED
    - Given original XML and rows with a node removed, diff returns 1 NodeChange with change_type=REMOVED
    - GameDevMergeResult.changed_nodes counts total changed, added_nodes counts added, removed_nodes counts removed

    GMERGE-02 (node-level operations):
    - apply_changes with MODIFIED NodeChange updates the correct element's attributes
    - apply_changes with ADDED NodeChange inserts a new element at the correct position
    - apply_changes with REMOVED NodeChange removes the element from the tree

    GMERGE-03 (attribute-level merge):
    - When one attribute changes and others stay the same, only the changed attribute is updated
    - When a new attribute is added, it appears in the output without affecting existing attributes
    - When an attribute is removed, it disappears without affecting sibling attributes

    GMERGE-04 (depth handling):
    - Diff correctly identifies changes in child elements (depth 2)
    - Diff correctly identifies changes in sub-child elements (depth 3)
    - Nodes beyond max_depth in original tree are preserved unchanged (not flagged as removed)

    GMERGE-05 (position preservation):
    - Output XML maintains the same element order as the original (modified elements stay in place)
    - Added elements are inserted at their intended position, not appended to end
    - Removed elements do not shift the positions of remaining elements incorrectly
  </behavior>
  <implementation>
    1. Create `tests/fixtures/xml/gamedev_modified.xml` -- copy of gamedev_sample.xml with: one Item attribute changed (Value="150" to Value="200"), one Item removed (Healing Potion), one new Item added at position 2, one nested Stats attribute changed.

    2. Create `server/tools/ldm/services/gamedev_merge.py` with:
       - ChangeType enum: UNCHANGED, MODIFIED, ADDED, REMOVED
       - AttributeChange dataclass: name, old_value, new_value
       - NodeChange dataclass: change_type, tag, position, depth, attribute_changes, children_changes
       - GameDevMergeResult dataclass: total_nodes, changed_nodes, added_nodes, removed_nodes, modified_attributes, changes list, output_xml bytes

       - `_reconstruct_tree_from_rows(rows: list[dict]) -> etree._Element`:
         Reconstruct the expected tree structure from DB rows using extra_data fields.
         Use depth to determine parent-child relationships. Build a virtual root, attach nodes by depth tracking.

       - `diff_trees(original: etree._Element, current_rows: list[dict], max_depth: int = 3) -> list[NodeChange]`:
         Parallel walk: iterate original tree elements (skipping root, comments, PIs) with same logic as parse_gamedev_nodes. For each original element at position P with tag T at depth D:
         - Find corresponding row by matching (position, tag, depth) -- rows are ordered by parse_gamedev_nodes order which matches document order
         - Compare attributes dict from extra_data against element.attrib
         - If no corresponding row exists -> REMOVED
         - Build AttributeChange for each differing attribute
         - Recurse into children (handle depth)
         Then check for rows with no corresponding original element -> ADDED

         CRITICAL: Match by sequential position in document order, not by attribute values. parse_gamedev_nodes iterates in document order, so row index maps directly to element iteration order.

         CRITICAL: Nodes in original tree beyond max_depth have no rows -> treat as UNCHANGED (pass through), NOT as REMOVED.

       - `apply_changes(root: etree._Element, changes: list[NodeChange]) -> None`:
         Apply in-place to original tree:
         - MODIFIED: find element by position, call elem.set() for changed attributes, del elem.attrib[name] for removed attributes
         - REMOVED: process in REVERSE position order to avoid index shifting, use elem.getparent().remove(elem)
         - ADDED: use parent.insert(position, new_elem) NOT SubElement (preserves document order)

       - `merge(original_xml: bytes, current_rows: list[dict], max_depth: int = 3) -> GameDevMergeResult`:
         Parse original_xml with lxml (recover=True), call diff_trees, call apply_changes, serialize with etree.tostring(pretty_print=True, xml_declaration=True, encoding from original). Return GameDevMergeResult with counts and output_xml.

       NO postprocessing pipeline -- attribute values written as-is (per research anti-pattern guidance).
       NO shared base class with TranslatorMergeService -- completely separate service.

    3. Create `tests/unit/ldm/test_gamedev_merge.py` with test classes:
       - TestDiffDetection: tests for GMERGE-01 (unchanged, modified, added, removed detection)
       - TestNodeOperations: tests for GMERGE-02 (apply add/remove/modify)
       - TestAttributeMerge: tests for GMERGE-03 (single attr change, attr add, attr remove)
       - TestDepthHandling: tests for GMERGE-04 (child changes, sub-child changes, beyond max_depth preservation)
       - TestDocumentOrder: tests for GMERGE-05 (output order matches original, insert position correct)

       Use gamedev_sample.xml as original fixture. Build current_rows programmatically (parse original, modify specific extra_data fields to simulate edits).
  </implementation>
