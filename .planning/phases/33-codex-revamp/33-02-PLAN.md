  <files>locaNext/src/lib/components/pages/CodexPage.svelte</files>
  <action>
Add a `Map` cache to CodexPage that stores loaded entities per tab so switching back to a previously visited tab is instant.

**Add cache state variable** (after existing state declarations):
```svelte
// Per-tab cache: Map<entityType, { entities: [], currentPage: number, hasMore: boolean }>
let tabCache = $state(new Map());
```

**Modify `selectTab`** to check cache before fetching:
```svelte
function selectTab(type) {
  activeTab = type;
  selectedEntity = null;

  // Check cache
  const cached = tabCache.get(type);
  if (cached) {
    entities = cached.entities;
    currentPage = cached.currentPage;
    hasMore = cached.hasMore;
    return; // No fetch needed
  }

  // No cache -- fetch from page 0
  currentPage = 0;
  hasMore = true;
  fetchEntityPage(type, 0);
}
```

**Update `fetchEntityPage`** to save results to cache after successful fetch. At the end of the try block, after setting `entities`, `hasMore`, and `currentPage`, add:
```svelte
// Update tab cache
tabCache.set(entityType, {
  entities: page === 0 ? batch : [...entities],
  currentPage: page,
  hasMore: data.has_more ?? (batch.length === PAGE_SIZE),
});
// Force reactivity on the Map
tabCache = new Map(tabCache);
```
Wait -- the entities variable is already updated by this point. Use the current `entities` value:
```svelte
tabCache.set(entityType, { entities, currentPage, hasMore });
tabCache = new Map(tabCache);
```
Place this AFTER the `currentPage = page;` line.

**Update `loadMore`** to also update cache for the active tab after each page load. This is already handled by the fetchEntityPage cache update above.

**Invalidate cache on batch generation complete.** In the `unsubComplete` handler where `fetchEntityPage(activeTab, 0)` is called, also clear the cache for that tab:
```svelte

tabCache = new Map(tabCache);
```
