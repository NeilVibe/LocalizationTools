Build Trigger File - LocaNext LIGHT Build System
=================================================

USAGE:
------
Add a line with the build command, then commit & push to main.
GitHub Actions will automatically build when it detects the new line.

FORMATS:
  Build LIGHT v[version]  → Build LIGHT installer (~100-150MB, no model)
  Build FULL v[version]   → Build FULL installer (~2GB, model bundled) [BLOCKED - LFS quota]

EXAMPLES:
  Build LIGHT v2511301200   → LIGHT installer, model downloads post-install
  Build FULL v2511301200    → FULL installer with bundled model (needs LFS quota)

WORKFLOW (VRS-Manager Protocol):
1. Update VERSION in version.py (single source of truth)
2. Run: python3 scripts/check_version_unified.py
3. If all green ✅, commit version changes
4. Add build command to this file
5. Commit and push to main
6. GitHub Actions will automatically build

LIGHT vs FULL:
- LIGHT: Small installer, model downloads during install from Hugging Face
- FULL: Large installer, model bundled (requires LFS bandwidth quota)

RECOMMENDED: Use LIGHT builds to avoid LFS costs!
